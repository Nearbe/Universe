    Уровень 1: Операторы цикла (движение по циклу)


    ┌──────────────────────┬────────────┬──────────────────────────────────────────────────────────┐
    │ Модуль               │ Функция    │ Как двигает по циклу                                     │
    ├──────────────────────┼────────────┼──────────────────────────────────────────────────────────┤
    │ branching.D          │ D(x) = 2x  │ Шаг вперёд по циклу: 0→1→2→...→9→0                       │
    ├──────────────────────┼────────────┼──────────────────────────────────────────────────────────┤
    │ branching.H          │ H(x) = x/2 │ Шаг назад по циклу: 9→8→7→...→0→9                        │
    ├──────────────────────┼────────────┼──────────────────────────────────────────────────────────┤
    │ division.safe_divide │ a:b        │ Выбор направления: b=Ω → D (вперёд), b=D(Id) → H (назад) │
    └──────────────────────┴────────────┴──────────────────────────────────────────────────────────┘


    Цикл — это D и H. Не абстракция. D = шаг вперёд. H = шаг назад.

    spine.py — это координатная система цикла. ridge_level(n) = n (номер узла). ridge_to_percentage(n) =
    sigmoid(n) × 100% (позиция в [-1, 1]).

    Все 20 модулей core — это один цикл. Не "разные модули". Не "комплиментарные представления". Один цикл в 20
    аспектах:

     1. Константы = состояния цикла
     2. Операторы = движение по циклу
     3. Координаты = позиция на цикле
     4. Представления = координаты цикла
     5. Метрики = расстояние на цикле
     6. Структуры = паттерны на цикле
     7. Динамика = переходы на цикле
