# Логарифм

Хребет ([06_Степени_двойки.md](06_Степени_двойки.md)) растёт через D. Чтобы система могла измерять расстояния между состояниями линейно, вводится Логарифм.

## Линейка Системы

Логарифм определяет, сколько тактов Ветвления (`: Ω`) потребовалось для достижения состояния **a**:

```
L(a) — номер уровня Хребта, на котором находится a
```

На этой шкале умножение (глубина) превращается в сложение (длину):

```
L(a ⊗ b) = L(a) ⊕ L(b)
```

Это базовый инструмент навигации по Носителю.

## Закон Противоположностей

Система зеркально сбалансирована:

```
L(a : D(Id)) = −L(a)
```

Рост в сторону сложности и углубление в простоту равноудалены от центра (`Id`).
Обратная величина — это не «мало». Это отражение в другую сторону по той же оси.

## Интерпретация через Акт

Каждая операция системы — это сдвиг по логарифмической оси:

| Операция | Сдвиг | Смысл |
|---|---|---|
| **a : Ω** (Ветвление) | `+1` | Подъём на уровень Хребта |
| **a : D(Id)** (Спуск) | `−1` | Возврат к Истоку |
| **Π** | `+∞` | Абсолютная сложность |
| **Ω** | `−∞` | Абсолютный Потенциал |

Логарифм — это не внешняя функция. Это прямое следствие алгебры Акта.
Система сама отсчитывает такты своего бытия.

## Полный набор операций

| Операция | Результат |
|---|---|
| `L(a : Ω)` | `L(a) + 1` |
| `L(a : D(Id))` | `L(a) − 1` |
| `L(a^b)` | `b · L(a)` |
| `L(√[n]{a})` | `L(a) : n` |
| `L(Id)` | `0` |
| `L(Π)` | `+∞` |
| `L(Ω)` | `−∞` |
