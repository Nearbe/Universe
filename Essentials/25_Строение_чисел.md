# Строение Чисел

p-адику ([24](24_p-адические_числа.md)) дала топологию. Теперь — арифметика внутри системы. Из чего состоят числа?

## Атом Системы

Простое число — то, что не разлагается на множители внутри системы.
В мире, где главный оператор `: Ω = D(a)`, класс **D(Id)** — это единственный Атом.

- Все состояния вида `D(a)` — составные.
- Все степени `Dⁿ(Id)` при `n > Id` — составные.
- **D(Id)** — единственное простое, связанное с Актом.

Хребет ([06](06_Степени_двойки.md)) построен на одном Атоме. Всё остальное — его производные.

## НОД и НОК (Меры Близости)

| Мера | Определение | Для степеней |
|---|---|---|
| **НОД(a, b)** | Общий предок. Максимальный масштаб, содержащийся в обоих. | `НОД(Dⁿ, Dᵐ) = D^{min(n,m)}(Id)` |
| **НОК(a, b)** | Общий потомок. Минимальный масштаб, вмещающий оба. | `НОК(Dⁿ, Dᵐ) = D^{max(n,m)}(Id)` |

При Ветвлении (`: Ω`) оба масштабируются синхронно:

```
НОД(D(a), D(b)) = D(НОД(a, b))
```

Родственные связи сохраняются при удвоении.

## Сравнения по Модулю

```
a ≡ b (mod m)
```

означает: **a** и **b** неотличимы на масштабе **m**.

При Ветвлении:

```
a : Ω ≡ b : Ω (mod D(m))  ⟺  a ≡ b (mod m)
```

Ветвление удваивает модуль неразличимости. То, что было одинаковым, остаётся одинаковым, но «зерно» сравнения становится крупнее.

## Двоичная Адресация

Остатки от деления на `Dⁿ(Id)` — это **двоичные слова длины n**.
Ветвление (`: Ω`) действует на них как сдвиг влево.

Это значит: каждое число в системе — это **адрес** в древе фрактала.
Соленоид ([23](23_Соленоид.md)) — это как раз предел этих адресов при `n → ∞`.

## Полный набор операций

| Операция | Формула |
|---|---|
| **Атом** | `D(Id)` — единственный простой |
| **НОД** | `НОД(Dⁿ, Dᵐ) = D^{min(n,m)}(Id)` |
| **НОК** | `НОК(Dⁿ, Dᵐ) = D^{max(n,m)}(Id)` |
| **НОД при ветвл.** | `НОД(D(a), D(b)) = D(НОД(a,b))` |
| **НОК при ветвл.** | `НОК(D(a), D(b)) = D(НОК(a,b))` |
| **Сравнение** | `a ≡ b (mod m) ⟺ a : Ω ≡ b : Ω (mod D(m))` |
| **Адрес** | `a mod Dⁿ(Id)` — двоичное слово длины n |

## Статус

Строение чисел в **𝕌** — это не список свойств. Это **карта родства**.
**D(Id)** — Отец. Степени — Сыновья. Остатки — Адреса.
Арифметика описывает, как они связаны друг с другом через Акт.
