# Анализ Данных (Реставрация Сигнала)

Данные — это не «факты». Это **отголоски** вибраций ([34](34_Вибрации.md)).

## Шум и Сигнал

В системе **𝕌** нет «мусора». Есть вибрации разных частот.
- **Сигнал:** Это низкочастотная, устойчивая вибрация (Баланс, `Δ ≈ Ω`).
- **Шум:** Это высокочастотный хаос (`λ > Ω`), который маскирует сигнал.

Анализ данных — это не «поиск закономерностей». Это **Фильтрация**.
Мы отбрасываем лишние слои Соленоида ([23](23_Соленоид.md)), чтобы добраться до **Ядра**.

## Методы

- **Кластеризация:** Поиск сгустков энергии. (Где узлы резонанса?)
- **Рецепторы:** Поиск аномалий. (Где система вышла из равновесия?)
- **Сжатие:** Поиск главного. (Какие доли данных дают основную массу?)

## Полный набор операций

| Операция | Смысл |
|---|---|
| **Фильтрация** | Отбросить шум → оставить сигнал |
| **Кластеризация** | Найти сгустки (узлы резонанса) |
| **Аномалии** | Найти отклонения от `Δ ≈ Ω` |
| **Сжатие** | Малая доля → основная масса |

## Физический Смысл

Аналитик данных — это **реставратор**.
Он снимает вековой слой пыли (шума), чтобы увидеть изначальный рисунок (Закон), который был заложен в Акте.
Данные не «говорят». Данные **молчат**, пока ты не настроишь свой приёмник на их частоту.
