# Бинарный поиск

## Определение

Алгоритм поиска элемента в отсортированном массиве, работающий за $O(\log n)$.

## Алгоритм

```
binary_search(A, target):
    left = 0, right = n - 1
    while left <= right:
        mid = (left + right) // 2
        if A[mid] == target: return mid
        elif A[mid] < target: left = mid + 1
        else: right = mid - 1
    return -1  # не найдено
```

## Сложность

- **Время:** $O(\log n)$ — на каждом шаге область поиска сокращается вдвое
- **Память:** $O(1)$ итеративно, $O(\log n)$ рекурсивно

## Требования

Массив должен быть отсортирован.

## Применения

- Поиск в словарях и базах данных
- Поиск ответа на монотонную функцию (бинарный поиск по ответу)
- Нижняя граница в дереве решений для поиска: $\Omega(\log n)$

## Обобщения

- **Двоичное дерево поиска:** динамическая структура, $O(\log n)$ вставка/удаление/поиск
- **B-дерево:** для дисковых структур (файловые системы, СУБД)
