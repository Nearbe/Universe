# Positional Encoding

## Определение

Способ добавления информации о позиции токена в последовательности, поскольку Transformer не имеет встроенного понятия порядка.

## Синусоидальное кодирование (Vaswani et al., 2017)

$$PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d}}\right)$$
$$PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d}}\right)$$

где $pos$ — позиция, $i$ — индекс признака, $d$ — размерность.

## Свойства

- Позволяет модели выучить относительные позиции (через тригонометрические тождества)
- Работает для произвольной длины последовательности (экстраполяция)

## Обучаемое позиционное кодирование

Вместо фиксированных функций — обучаемые векторы позиций:
- GPT: обучаемые embedding'и позиций
- BERT: то же, плюс token type embeddings
- T5: кодирование относительной позиции (bias в attention)

## Альтернативы

- **Rotary Position Embedding (RoPE):** вращение Q и K в attention
- **ALiBi:** линейный bias в attention scores
- **NoPE:** использование только attention без явного кодирования
