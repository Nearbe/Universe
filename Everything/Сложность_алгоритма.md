# Сложность алгоритма

## Определение

Функция, описывающая зависимость ресурсов (времени или памяти), потребляемых алгоритмом, от размера входа $n$.

## O-нотация (Большое O)

$f(n) = O(g(n))$ означает, что существует константа $C$, такая что $f(n) \leq C \cdot g(n)$ для достаточно больших $n$.

Описывает **верхнюю границу** роста функции.

## Классы сложности

| Обозначение | Название | Пример |
|---|---|---|
| $O(1)$ | Константная | Доступ к элементу массива |
| $O(\log n)$ | Логарифмическая | Бинарный поиск |
| $O(n)$ | Линейная | Поиск в несортированном массиве |
| $O(n \log n)$ | Линейно-логарифмическая | Быстрая сортировка |
| $O(n^2)$ | Квадратичная | Сортировка пузырьком |
| $O(2^n)$ | Экспоненциальная | Перебор подмножеств |
| $O(n!)$ | Факториальная | Задача коммивояжёра (точный метод) |

## P и NP

- **P:** задачи, решаемые за полиномиальное время
- **NP:** задачи, решение которых проверяется за полиномиальное время

Вопрос $P = NP$ — одна из задач тысячелетия.
