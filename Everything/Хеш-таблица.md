# Хеш-таблица

## Определение

Структура данных, обеспечивающая вставку, поиск и удаление за среднее $O(1)$.

## Принцип

Ключ $k$ отображается в индекс массива через хеш-функцию:
$$\text{index} = h(k) \bmod m$$

где $m$ — размер таблицы.

## Хеш-функция

Требования:
- **Детерминированность:** один ключ → один хеш
- **Равномерность:** хеши распределены равномерно
- **Эффективность:** быстрое вычисление

Примеры:
- Целые: $h(k) = k$
- Строки: $h(s) = \sum s_i \cdot 31^i \bmod 2^{32}$

## Разрешение коллизий

**Метод цепочек:** каждый индекс — связный список элементов с одинаковым хешем.

**Открытая адресация:** при коллизии ищется следующая свободная ячейка (линейное, квадратическое зондирование, двойное хеширование).

## Сложность

- **Средняя:** $O(1)$ вставка/поиск/удаление
- **Худшая:** $O(n)$ при множественных коллизиях

## Применения

Словари (dict в Python), кеш-таблицы, базы данных, множества (set).
