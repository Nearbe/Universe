# Дерево двоичного поиска

## Определение

Двоичное дерево, в котором для каждой вершины:
- Все ключи левого поддерева < ключ вершины
- Все ключи правого поддерева > ключ вершины

## Операции

**Поиск:**
```
search(node, key):
    if node is null or node.key == key: return node
    if key < node.key: return search(node.left, key)
    return search(node.right, key)
```

**Вставка:** спуск до листа, добавление нового узла.

**Удаление:**
- Лист: просто удалить
- Один ребёнок: заменить ребёнка на родителя
- Два ребёнка: заменить преемником (минимум правого поддерева)

## Сложность

| Операция | Средняя | Худшая |
|---|---|---|
| Поиск | $O(\log n)$ | $O(n)$ |
| Вставка | $O(\log n)$ | $O(n)$ |
| Удаление | $O(\log n)$ | $O(n)$ |

Худший случай — вырожденное дерево (цепочка).

## Сбалансированные варианты

- **AVL-дерево:** балансировка по высоте, $O(\log n)$ гарантированно
- **Красно-чёрное дерево:** балансировка по цвету, используется в `std::map`, `std::set`
- **B-дерево:** для дисковых структур
