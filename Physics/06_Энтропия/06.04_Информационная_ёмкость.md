# Информационная ёмкость и каналы связи

## Канал связи с шумом

### Модель канала

Рассматривая сеть ходонов как канал связи:

- **Входной сигнал**: s(t) — модулирует параметр Δ
- **Шум**: n(t) — обусловлен флуктуациями Δ
- **Выходной сигнал**: r(t) = s(t) + n(t)

### Отношение сигнал/шум

```text
SNR = σ_s² / σ_n²
```

где:

- σ_s² — дисперсия сигнала
- σ_n² — дисперсия шума, σ_n² ∝ σ(Δ)

### Пропускная способность

По формуле Шеннона-Хартли:

```text
C = B · log₂(1 + SNR)
```

где B — полоса пропускания канала.

## SNR в терминах IME

### Связь с синхронизацией

Отношение сигнал/шум связано с синхронизацией:

```text
SNR ∝ 𝒮 / (1 - 𝒮)
```

где 𝒮 = 1 - σ(Δ)·scale.

### Вывод

При высокой синхронизации (𝒮 → 1):

```text
SNR → ∞
```

При низкой синхронизации (𝒮 → 0):

```text
SNR → 0
```

### Явная формула

```text
SNR = [1 - σ(Δ)·scale] / [σ(Δ)·scale]
```

## Оптимальное кодирование

### Гауссовское распределение

Для максимальной пропускной способности при заданной средней мощности следует использовать гауссовское распределение сигналов.

### В терминах IME

Это соответствует гауссовскому распределению параметра Δ по сети:

```text
p(Δ) = (1/√(2πσ²)) · exp[-(Δ - ⟨Δ⟩)² / (2σ²)]
```

### Оптимальное стандартное отклонение

Максимальная информационная ёмкость на единицу времени достигается при:

```text
σ(Δ)_opt = σ_s / √(e^(2·C/B) - 1)
```

где C/B — спектральная эффективность.

## Информация и работа

### Принцип Ландауэра

Стирание одного бита информации требует минимальной работы:

```text
W_min = k_B·T·ln(2)
```

### В терминах IME

Изменение параметра Δ на величину, соответствующую стиранию одного бита, требует работы:

```text
ΔW = k_B·T·ln(2) · |∂Δ/∂I|
```

### Чувствительность Δ к информации

Из соотношения I ∝ e^(Δ/2) для фиксированной площади:

```text
∂I/∂Δ = (1/2) · I
```

```text
∂Δ/∂I = 2 / I
```

### Работа для изменения информации

```text
ΔW = 2·k_B·T·ln(2) / I
```

### Физический смысл

Работа, необходимая для изменения информации, **уменьшается** с ростом информационного содержимого системы.

Это эффект **информационной термодинамики**:

- Богатые информацией системы легче модифицировать
- Бедные информацией системы требуют больше работы

## Множественные каналы

### Параллельные каналы

Для N параллельных каналов с пропускными способностями C_i:

```text
C_total = Σ C_i
```

### В терминах IME

Сеть ходонов может поддерживать множественные параллельные каналы:

- Разные частотные полосы
- Разные пространственные моды
- Разные поляризации

### Иерархия каналов

```text
C_hierarchy = C_1 + C_2/2 + C_3/4 + ...
```

где каждый последующий уровень вносит меньший вклад.

## Пропускная способность сети ходонов

### Для сети из N ходонов

Максимальная пропускная способность:

```text
C_network = N · B · log₂(1 + SNR)
```

### С учётом синхронизации

При синхронизации 𝒮:

```text
C_network = N · B · log₂(1 + 𝒮/(1-𝒮))
```

### При высокой синхронизации

При 𝒮 → 1:

```text
C_network → N · B · log₂(1/σ(Δ)·scale)
```

Логарифмически растёт с уменьшением σ(Δ).

## Квантовые каналы

### Квантовая пропускная способность

Для квантового канала с запутанностью:

```text
C_quantum = log₂(d) + E_N
```

где:

- d — размерность пространства состояний
- E_N — информация запутанности (entanglement)

### В терминах IME

Запутанность связана с корреляциями Δ между ходонами:

```text
E_N ∝ ln(cosh(Δ_corr))
```

где Δ_corr — коррелированное значение Δ.
