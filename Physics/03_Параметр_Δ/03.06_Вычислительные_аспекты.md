# Вычислительные аспекты параметра Δ

## Устойчивые алгоритмы вычисления

### Проблема прямого вычисления

Наивное вычисление через отношение:

```text
Δ = ln(|Re|/|Im|)
```

приводит к проблемам при:

- Im → 0: деление на ноль
- Re → 0: отношение равно нулю, ln(0) = -∞
- Re = Im = 0: неопределённость 0/0

### Стабильный алгоритм

Предпочтительно вычисление как **разности логарифмов**:

```text
function compute_delta(Re, Im):
    absRe = abs(Re)
    absIm = abs(Im)

    if absIm == 0:
        if absRe == 0:
            return NaN  # истинная неопределённость 0/0
        else:
            return +inf  # сингулярность Im=0

    if absRe == 0:
        return -inf  # тёмная энергия Re=0

    return log(absRe) - log(absIm)  # избегаем деления
```

## Обработка специальных значений IEEE 754

### Стандарт плавающей точки

Стандарт IEEE 754 for floating-point arithmetic корректно обрабатывает:

| Операция | Результат |
|----------|-----------|
| log(0) | -∞ |
| log(+∞) | +∞ |
| ∞ - ∞ | NaN |
| конечное - ∞ | -∞ |
| ∞ - конечное | +∞ |

### Таблица специальных случаев

| Re | Im | Вычисление | Результат |
|----|-----|------------|-----------|
| 0 | 0 | log(0) - log(0) | NaN |
| ≠0 | 0 | log( | Re |
| 0 | ≠0 | (-∞) - log( | Im |
| ∞ | конечное | +∞ - конечное | +∞ |
| конечное | ∞ | конечное - (+∞) | -∞ |
| ∞ | ∞ | +∞ - (+∞) | NaN |

### Рекомендации по реализации

1. **Всегда использовать разность логарифмов**, а не логарифм отношения
2. **Проверять на NaN** после вычисления — это корректный результат для Re = Im = 0
3. **Обрабатывать ±∞** как валидные физические состояния

## Численная точность

### Потеря точности при близких значениях

При |Re| ≈ |Im| (Δ ≈ 0) может происходить потеря значащих цифр из-за вычитания близких величин.

### Решение

Использовать повышенную точность (double вместо float) или специализированные функции:

```text
// Для малых Δ использовать разложение в ряд
if abs(|Re| - |Im|) < ε:
    ratio = |Re|/|Im|
    Δ = (ratio - 1) - (ratio - 1)²/2 + (ratio - 1)³/3 - ...
```

## Вычисление в пакетном режиме

### Для массивов данных

При обработке больших массивов (например, сети ходонов):

```text
function compute_delta_array(Re[], Im[]):
    Δ[] = array of same size

    for i in range(len(Re)):
        Δ[i] = compute_delta(Re[i], Im[i])

    return Δ
```

### Векторизованная реализация

В NumPy или аналогичных библиотеках:

```python
import numpy as np

def compute_delta_numpy(Re, Im):
    with np.errstate(divide='ignore', invalid='ignore'):
        Δ = np.log(np.abs(Re)) - np.log(np.abs(Im))
    return Δ
```

## Статистические вычисления

### Среднее значение

```text
⟨Δ⟩ = (1/N) Σ Δᵢ
```

### Стандартное отклонение

```text
σ(Δ) = √[(1/N) Σ (Δᵢ - ⟨Δ⟩)²]
```

Это критически важно для вычисления синхронизации 𝒮.

### Гистограмма распределения

Для анализа распределения Δ по сети:

```text
function compute_delta_histogram(Δ[], bins):
    histogram = zeros(bins)
    bin_width = (Δ_max - Δ_min) / bins

    for Δᵢ in Δ:
        bin_index = floor((Δᵢ - Δ_min) / bin_width)
        histogram[bin_index] += 1

    return histogram
```

## Оптимизация производительности

### Кэширование логарифмов

При многократных вычислениях с теми же значениями:

```text
log_cache = {}

function cached_log(x):
    if x not in log_cache:
        log_cache[x] = log(x)
    return log_cache[x]
```

### Приближённые вычисления

Для приложений реального времени можно использовать приближения:

```text
// Приближение логарифма для x ≈ 1
log(x) ≈ (x - 1) - (x - 1)²/2 + (x - 1)³/3
```

## Мониторинг вычислений

### Отслеживание специальных значений

```text
function monitor_delta(Δ[]):
    n_nan = count(isnan(Δ))
    n_inf = count(isinf(Δ))
    n_finite = count(isfinite(Δ))

    print(f"NaN: {n_nan}, Inf: {n_inf}, Finite: {n_finite}")
```

### Предупреждения

Генерировать предупреждения при:

- Доля NaN > 1% (возможная ошибка в данных)
- Доля ±∞ > 10% (система близка к сингулярности)
- σ(Δ) > 6.0 (синхронизация ниже порога сознания)
