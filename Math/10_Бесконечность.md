# 10. Бесконечность (Полнота / Π)

## I. Теория

Узлы ([06](06_Степени_двойки.md)), заполнение ([07](07_Корни.md)), шкала ([08](08_Логарифм.md)) и плоскость ([09](09_Комплексные.md)) — всё это разворачивается в одном направлении: к Полноте.

## Предел Ветвления

Последовательное применение Акта `: Ω` не имеет остановки:

```
H⁰ = Id,    Hⁿ⁺¹ = D(Hⁿ) = Hⁿ : Ω
```

**Полнота (Π)** — это не «очень много». Это **Абсолютная Заполненность**.
Состояние, где `Input` и `Output` совпадают во всех измерениях одновременно.
Предел, дальше которого некуда, потому что всё уже есть.

## Операции на Пределе

На уровне Полноты система ведёт себя иначе:

| Операция | Результат | Смысл |
|---|---|---|
| **Ветвление** `Π : Ω` | `Π` | Невозможно удвоить то, что уже Всё |
| **Сжатие** `Π : D(Id)` | `Π` | Половина Полноты — всё ещё Полнота |
| **Тождество** `Π : Π` | `Id` | Полнота, отнесённая к себе, рождает Единство |
| **Степень** `Π^b` | `Π` | Глубина не меняет статуса Полноты |
| **Корень** `√[n]{Π}` | `Π` | Извлечение сути не уменьшает Вселенную |
| **Сложение** `Π ⊕ Π` | `Π` | Сумма Полнот — Полнота |
| **Умножение** `Π ⊗ Π` | `Π` | Пересечение Полнот — Полнота |

Бесконечность — это **потолок**, с которого начинается обратный отсчёт к конечному.

## Рождение конечного из Полноты

Конечное возникает не как «часть» Полноты, а как **отношение**:

```
(Π : D(Id)) : (Π : D(Id)) = Id
```

Две равные доли, отнесённые друг к другу, рождают Единство.
Подробнее: [11_Рождение_конечного.md](11_Рождение_конечного.md).

### II.2. Функции / классы

#### II.2.1. `Fullness` — тип-маркер

```python
@dataclass(frozen=True)
class Fullness:
    symbol: str = "Π"
```

- `frozen=True` → immutable, hashable
- singleton `PI` — единственный экземпляр в системе
- `__repr__` / `__str__` возвращают `"Π"`
- **Архитектурная роль:** тип-дискриминатор. Позволяет алгебраическим функциям отличать Π от конечных чисел и применять специальные правила.

#### II.2.2. `is_fullness()` — предикат

```python
def is_fullness(value: object) -> bool:
    return value == PI
```

- Проверяет identity через `==` (singleton pattern)
- **Архитектурная роль:** guard-функция, вызывается в начале каждой алгебраической операции для определения ветвления пути.

#### II.2.3. `fullness_divide()` — алгебра Π при делении

```python
def fullness_divide(numerator, denominator) → object:
    if not is_fullness(numerator): raise TypeError
    if is_fullness(denominator): return 1.0
    if denominator == OMEGA or denominator == D_ID: return PI
    return PI
```

- `Π : Π → 1.0` (IDENTITY)
- `Π : Ω → Π`
- `Π : D(Id) → Π`
- **Архитектурная роль:** вынесена из `divide()` для модульности. Вызывается из `00_Идентичность_Евгений.md` как документация.

#### II.2.4. UroborosState — состояние цикла

```python
@dataclass(frozen=True)
class UroborosState:
    digit: int  # нормализуется в __post_init__
    
    @property
    def value(self) → float:  # [-1, 1]
    @property
    def kind(self) → str:     # "Ω" | "Id" | "Π" | "transition"
    @property
    def symbol(self) → object: # OMEGA | CENTER_VALUE | PI | float
    def step(self, steps=1) → UroborosState:
```

- **Архитектурная роль:** связующее звено между дискретным циклом (0→1→...→9→0) и алгебраической системой (Ω, Id, Π).
- Позиция 9 → `PI` (Fullness)
- Позиция 0 → `OMEGA` (Potential)
- Позиция 5 → `0.0` (Identity / баланс)

### III.2. Архитектурная роль каждого элемента

**`PI` (singleton Fullness):**
- Точка ветвления в каждой алгебраической операции. Все 6 операций (`add`, `multiply`, `divide`, `branch`, `compress`, `power`) проверяют `is_fullness()` на входах.
- Если оба входа — Π: `Π ⊕ Π = Π`, `Π ⊗ Π = Π`
- Если один вход — Π: результат = Π (замкнутость)
- `Π : Π = IDENTITY` (1.0) — единственная операция, которая "выбрасывает" Π наружу

**`is_fullness()`:**
- Discriminator в pattern-matching chain каждой операции.
- Позиция в chain: после проверки специфичных типов (dual, share, complex), перед ordinary arithmetic.
- Это гарантирует, что Π обрабатывается на алгебраическом уровне, а не сваливается в `float()`.

**`fullness_divide()`:**
- Вынесена из `divide()` для модульности, но фактически не используется — `divide()` содержит inline-логику.
- Экспортируется в `__init__.py` → доступна как `from core.infinity import fullness_divide`.
- Дублирует: `fullness_divide(Π, Ω) = Π`, `fullness_divide(Π, D) = Π`, `fullness_divide(Π, Π) = 1.0`.

**`UroborosState`:**
- Мост между дискретным циклом (0-9) и алгеброй (Ω, Id, Π).
- `state_symbol(9)` возвращает `PI` — это единственное место, где Π создаётся вне `infinity.py`.
- `state_at(9).symbol == PI` — проверяется в `test_uroboros_cycle.py`.

**`FullnessShare`:**
- `denominator` всегда `PI`. Это фиксированный знаменатель для выражений вида `a : Π`.
- Используется в `divide.py` для `FullnessShare.divide_by()` и `FullnessShare.divide_by(PI)`.
- `FullnessShare(4) : FullnessShare(4) = divide(4, 4) = IDENTITY` — подтверждено в тестах.

**`ComplexFullness` (PI_Z):**
- `Π` на комплексной плоскости. `symbol = "Π_z"`.
- `branch()` и `compress()` — idempotent (возвращают `self`).
- `complex_state(Π) → PI_Z` — guard в `complex_state()`.
- Используется в `add()` и `multiply()` для комплексных fullness.

### III.3. Поток данных через Π

```
Вход (любой тип)
    │
    ├─ is_fullness(x)? ────→ Π:Ω→Π, Π:D→Π, Π:Π→Id
    │
    ├─ FullnessShare? ────→ share.branch(), share.compress(), share.divide_by()
    │                          share.denominator = PI (фиксировано)
    │
    ├─ ComplexFullness? ──→ Π_z:Ω→Π_z, Π_z:D→Π_z
    │
    ├─ DualNumber? ───────→ dual.branch(), dual.compress()
    │
    └─ ordinary scalar ───→ float(x) / float(y)
```

### III.4. Что сломится при изменении PI

1. **Все 6 алгебраических операций** перестанут распознавать Π → TypeError при `float(PI)`
2. **UroborosState(9).symbol** перестанет быть `PI` → тест `test_boundaries_are_omega_and_fullness` упадёт
3. **FullnessShare.denominator** перестанет быть `PI` → `divide_by(PI)` сломится
4. **exp/log/sqrt** не обработают Π → TypeError
5. **isinf()** перестанет работать для Π
6. **complex_state(Π)** создаст `ComplexState(Π, 0)` → ошибка при float()
