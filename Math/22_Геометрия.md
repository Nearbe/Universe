# 22. Геометрия

## I. Теория

Расстояние между состояниями в 𝕌 определяется метрикой:

```
d(a, b) = |a ⊖ b|
```

При действии Ветвления (`: Ω`):

```
d(a : Ω, b : Ω) = |D(a) ⊖ D(b)| = D(d(a, b))
```

Расстояние ветвится. Это **гомотетия** с классом масштабирования **D(Id)**.
Геометрия системы **фрактальна**: каждый объект существует на всех уровнях иерархии одновременно, меняя только размер.

При Ветвлении объём в **k**-мерном пространстве масштабируется как `Dᵏ(Id)`:

| Объект | При `: Ω` | Множитель |
|---|---|---|
| Длина (1D) | `L → D(L)` | `D(Id)` |
| Площадь (2D) | `S → D²(S)` | `D²(Id)` |
| Объём (3D) | `V → D³(V)` | `D³(Id)` |

**Размерность пространства проявляется в степени масштабирования.**

Кривизна для кривой `y = f(x)`:

```
κ = |f''(x)| : (Id ⊕ f'(x)²)^{3 : D(Id)}
```

При `: Ω` кривизна **уменьшается**: `κ → κ : D(Id)`.

Комплексная плоскость:

```
z = x + i·y
|z| = √(x² ⊕ y²)
(x + i·y) : Ω = D(x) + i·D(y) = D(z)
```

Фаза (угол) сохраняется. Меняется только радиус.

Полнота ([10_Бесконечность.md](10_Бесконечность.md)) — не точка в пространстве, а **горизонт**.
Все лучи уходят к **Π**, но `Π : Π = Id` — Полнота равна самой себе.

### II.2. Геометрические объекты

| Класс | Модуль | Поля | Методы |
|-------|--------|------|--------|
| `Segment` | `u_geometry.py` | `start: float`, `end: float` | `length()`, `branch()`, `compress()` |
| `Circle` | `u_geometry.py` | `center: ComplexState`, `radius: float` | `branch()`, `compress()` |
| `Ball` | `u_geometry.py` | `center: ComplexState`, `radius: float` | `branch()`, `compress()` |

Все три — `@dataclass(frozen=True)` (неизменяемые).

**Segment:**
- `length()` → `distance(start, end)` = `|start − end|`
- `branch()` → `Segment(D(start), D(end))` = `Segment(start·D_ID, end·D_ID)`
- `compress()` → `Segment(H(start), H(end))` = `Segment(start/D_ID, end/D_ID)`

**Circle / Ball:**
- `branch()` → центр: `center.branch()` (умножение Re/Im на `D_ID`), радиус: `D(radius)`
- `compress()` → центр: `center.compress()` (деление Re/Im на `D_ID`), радиус: `H(radius)`
- При ветвлении/сжатии центра — `TypeError` если результат не `ComplexState`

### II.3. Функции расстояния и меры

| Функция | Подпись | Формула | Зависимости |
|---------|---------|---------|-------------|
| `distance(a, b)` | `(float, float) → float` | `abs(a − b)` | — |
| `branched_distance(a, b)` | `(object, object) → float` | `d(D(a), D(b))` | `branch`, `distance` |
| `compressed_distance(a, b)` | `(object, object) → float` | `d(H(a), H(b))` | `compress`, `distance` |
| `branch_measure(measure, dimension)` | `(object, int) → object` | `measure × D^dimension(Id)` | `power`, `D_ID` |
| `compress_measure(measure, dimension)` | `(object, int) → object` | `measure / D^dimension(Id)` | `power`, `D_ID` |
| `complex_radius(value)` | `(object) → object` | `√(x² + y²)` | `complex_state`, `sqrt` |

`branched_distance` и `compressed_distance` — коммутативны с `branch`/`compress`:
`branched_distance(a, b) = branch_measure(distance(a, b), dimension=1)`.

### II.4. Функции кривизны

| Функция | Подпись | Формула | Зависимости |
|---------|---------|---------|-------------|
| `curvature(first_derivative, second_derivative)` | `(object, object) → object` | `|f''| / (1 + f'²)^(3/D_ID)` | `power`, `abs` |
| `compressed_curvature(value)` | `(object) → object` | `H(κ) = κ / D_ID` | `compress` |

Формула кривизны в `curvature`:
```python
denominator = power(1.0 + fd * fd, CURVATURE_EXPONENT)  # (1 + f'²)^1.5
return abs(sd) / denominator  # |f''| / denom
```

### II.5. Функции членства

| Функция | Подпись | Логика | Зависимости |
|---------|---------|--------|-------------|
| `is_on_circle(point, circle)` | `(object, Circle) → bool` | `complex_radius(point − center) == radius` | `_relative_complex`, `complex_radius` |
| `is_inside_ball(point, ball)` | `(object, Ball) → bool` | `complex_radius(point − center) ≤ radius` | `_relative_complex`, `complex_radius` |

`_relative_complex(point, center)` → `ComplexState(point.real − center.real, point.imag − center.imag)`.
Бросает `TypeError` если `point` не является конечным комплексным состоянием.

### II.6. Вращение

| Функция | Подпись | Формула | Зависимости |
|---------|---------|---------|-------------|
| `rotate_z(value, angle)` | `(object, object) → object` | `z · e^(iθ)` | `complex_state`, `ComplexState.rotate` |

`ComplexState.rotate(angle)` использует `cos`/`sin` из `src/core/transcendental/`:
```python
cosine = cos(angle), sine = sin(angle)
return ComplexState(
    real * cosine - imag * sine,
    real * sine + imag * cosine
)
```

### II.8. Что нужно реализовать

1. **Расстояния для не-скалярных типов**: `distance()` принимает только `float/int`. Нет обёрток для `ComplexState`, `DualNumber`, `ParticipationRatio`.
2. **Метрики для k>1**: `branch_measure` и `compress_measure` реализованы, но нет функций для площади/объёма (2D/3D меры).
3. **Эллипс и параболу**: аналогично Circle/Ball.
4. **Дуга/сектор**: геометрические примитивы на комплексной плоскости.
5. **Геометрия SpineLevel**: `SpineLevel` имеет `branch()`/`compress()`/`scale()`/`root()`/`value()`/`level()`, но нет геометрических операций (расстояние между уровнями, мера уровня).
6. **Геометрия FullnessShare**: нет геометрических операций для долей полноты.
7. **Параметрические кривые**: сплайны, соленоид (23_Соленоид.md).
8. **Интеграция с calculus**: `curvature()` принимает скалярные производные, нет связи с `branched_derivative`/`compressed_derivative`.

## III. Зачем

**Инженерная аргументация:**

1. **Геометрия как слой масштабирования.** `branch_measure`/`compress_measure` — это утилиты для трансформации метрических величин при переходе между уровнями хребта. Без них невозможно корректно сравнивать объекты на разных глубинах рекурсии.

2. **Геометрические примитивы (Segment/Circle/Ball) нужны для:**
   - Определения областей в комплексной плоскости (принятие решений, классификация состояний)
   - Визуализации (рендереры в `src/renderers/` используют эти структуры)
   - Пространственных запросов в states-системе (поиск ближайших состояний, проверка принадлежности)

3. **Кривизна** — мост между дифференциальным исчислением и геометрией. Позволяет измерять «изгиб» траектории в пространстве состояний. Используется в анализе динамических систем.

4. **Вращение `rotate_z`** — базовая операция для циклических процессов (Great Cycle of Uroboros, фазовые переходы). Сохраняет норму, меняет только фазу.

5. **Зависимости от core:** геометрия не содержит собственной алгебры — все операции делегируются:
   - `branch`/`compress` — масштабирование
   - `power` — степенные преобразования
   - `sqrt` — извлечение корня для радиуса
   - `cos`/`sin` — тригонометрия для вращения
   - `ComplexState` — представление комплексных состояний
