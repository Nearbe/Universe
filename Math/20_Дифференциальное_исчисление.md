# 20. Дифференциальное Исчисление

## I. Теория

Квадратные уравнения ([19](19_Квадратные_уравнения.md)) показали нелинейность. Но система не статична. Она непрерывно меняется. Нужно измерить **скорость изменения** состояния.

### Производная через Дуальные Числа

Дуальное число ([14](14_Дуальные_числа.md)) хранит и Форму (**x**), и Потенциал Роста (**v**):

```
Z = x + v·ε,    где ε² = Ω
```

Для любой функции **f**, применённой к дуальному числу:

```
f(x + v·ε) = f(x) + f'(x)·v·ε
```

Коэффициент при **ε** после применения функции — это и есть **производная**, умноженная на скорость **v**.
Таким образом, производная `f'(x)` извлекается автоматически, как только система вычисляет `f(Z)`.

### Связь с Ветвлением

Параметр **t** — это число тактов Ветвления (`: Ω`).
При малом сдвиге **dt**:

```
x(t ⊕ dt) ≈ x(t) ⊕ x'(t)·dt
```

В нашей системе `x(t) = Dᵗ(Id)`. Тогда:

```
dx/dt = L(D(Id)) · Dᵗ(Id) = L(D(Id)) · x
```

Скорость роста пропорциональна текущему состоянию. Это фундаментальное свойство экспоненциальной природы Акта.

### Производные и Масштаб

Как ведёт себя производная при Ветвлении (`x → D(x)`)?
Если `g(x) = f(D(x))`, то:

```
g'(x) = D(Id) · f'(D(x))
```

Производная масштабируется вместе с функцией. Это **дифференциальная форма масштабной инвариантности**. Закон изменения не ломается при ветвлении системы, он просто становится быстрее.

### Таблица Скоростей

| Функция `f(x)` | Скорость `f'(x)` | Смысл |
|---|---|---|
| `x^b` | `b · x^(b : D(Id))` | Степенной рост |
| `D(x)` | `L(D(Id)) · D(x)` | Собственная функция (скорость ∝ функции) |
| `L(x)` | `Id : x` | Замедление по мере роста |
| `sin x` | `cos x` | Вращение (сдвиг фазы) |

### Старшие Производные

Вторая производная `f''(x)` — это скорость изменения скорости (ускорение).
Для `D(x)`:

```
f''(x) = L(D(Id))² · D(x)
```

Ускорение также пропорционально состоянию. Система разгоняется сама себя.

### Полный набор операций

| Операция | Формула |
|---|---|
| **Через дуальные** | `f(x + vε) = f(x) + f'(x)·v·ε` |
| **Ветвление** | `g(x) = f(D(x)) ⟹ g'(x) = D(Id)·f'(D(x))` |
| **Сжатие** | `h(x) = f(x : D(Id)) ⟹ h'(x) = f'(x : D(Id)) : D(Id)` |
| **2-я производная** | `f''(x)` — ускорение |
| **Правило цепи** | `(f∘g)'(x) = f'(g(x)) · g'(x)` |

### Интегральные операции

| Операция | Формула |
|---|---|
| **Неопределённый интеграл** | `∫ f(x)dx = F(x) ⊕ C` |
| **Определённый интеграл** | `∫ₐᵇ f(x)dx = F(b) ⊖ F(a)` |
| **Интеграл ветвления** | `∫ f(D(x))dx = (∫ f(u)du) : D(Id)` |
| **Интеграл сжатия** | `∫ f(x:D(Id))dx = (∫ f(u)du) ⊗ D(Id)` |
| **Интеграл по лог-глубине** | `f(D^ℓ)·L(D(Id))·D^ℓ` |

## III. Зачем

### Инженерная аргументация

**Модуль `core.calculus`** — это слой измерения динамики состояний. Все производные и интегралы строятся на `DualNumber` и извлекают скрытую компоненту (`velocity`) через дуальную арифметику.

**Зависимости от `core.calculus`:**
- `tests/test_differential_calculus.py` — 7 тестов: производные, ветвление, сжатие, цепь, ускорение
- `tests/test_integral_calculus.py` — 7 тестов: антиинтегралы, определённый интеграл, ветвление/сжатие, лог-глубина
- Никакие модули `src/` напрямую не импортируют `core.calculus` — это чистый математический слой, не имеющий циклических зависимостей

**Зависимости от `core.dual`:**
- `dual_func`, `dual_power`, `dual_form` — обёртки обратной совместимости, возвращают `(form, velocity)` tuple из `DualNumber`
- Ни один модуль `src/` и ни один тест напрямую не импортирует `core.dual` — обёртки доступны через `core.dual.__init__`

**Зависимости от `core.states`:**
- `DualNumber` — фундаментальный тип для всех производных. `differential_state()` возвращает `DualNumber`
- `UFunction` — функциональный обёрточный тип с методами `on_branch()` / `on_compress()` / `compose()`
- `SpineLevel` — используется в `integral_on_log_depth()` для взвешивания по лог-глубине
- `core.states.__init__` экспортирует все три типа публично

**Зависимости от `core.transcendental`:**
- `exp()` — используется в `ln()` (масштабирующая редукция), `exp'(x) = exp(x)` (нужен `exp_derivative`)
- `ln()` — используется в `L()` (логарифмическая глубина), `sqrt()` (редукция через `ln(1/x)`)
- `sin()` / `cos()` — используются в `periodic_function()` (UFunction), `exp(i·k·x)` (комплексная плоскость)
- `log2()` — используется в `L()` для вычисления `L(x) = log2(|x|)`
- `pi()`, `e()`, `half_pi()`, `two_pi()` — константы для всех трансцендентных функций
- `isinf()` — используется в `sqrt()` для обработки Π

**Зависимости от `core.algebra`:**
- Все функции calculus используют `add`, `multiply`, `branch`, `compress`, `divide`, `power` из `core.algebra`
- `DUAL_BASIS_SIZE`, `SCALAR_BASIS_SIZE` — константы из `BranchingOperator`

**Зависимости от `core.constants`:**
- `OMEGA = 0.0` — ε², потенциал, значение по умолчанию velocity
- `D_ID = 2.0` — класс ветвления, используется в `branched_derivative`, `compressed_derivative`, `branching_growth_velocity`

**Зависимости от `core.infinity`:**
- `PI` / `Fullness` — `exp()` и `ln()` возвращают `PI` для Π-входа
- `is_fullness()` — используется в `exp()`, `ln()`, `sqrt()`, `L()`

**Зависимости от `core.operators.L`:**
- `L()` — логарифмическая глубина, используется в `branching_growth_velocity()` и `integral_on_log_depth()`
- Обрабатывает Ω (`LOG_NEGATIVE_INFINITY`), Π (`LOG_POSITIVE_INFINITY`), SpineLevel, векторы

**Что сломится при изменении:**
- Изменение `DualNumber.apply()` сломает все производные (все извлекают velocity через дуальное расширение)
- Изменение `D_ID` сломает `branched_derivative`, `compressed_derivative`, `branching_growth_velocity`
- Изменение `SpineLevel.value()` сломает `integral_on_log_depth`
- Изменение `exp()` сломает `ln()` (масштабирующая редукция) и `periodic_function` (через `cos`/`sin`)
- Удаление `dual_func`/`dual_power`/`dual_form` не сломает ничего внутри проекта (не используются), но ломает внешнюю совместимость
