# 16. Функции

## I. Теория

Функция в системе — это **закон преобразования состояния**:

```
f : 𝕌 → 𝕌
```

Каждому входу **x** она сопоставляет выход `f(x)`, полученный через допустимые операции системы.

Класс функции определяется тем, как функция реагирует на Акт Ветвления (`: Ω`).

### Степенные (Масштабные)

```
f(x) = x^b
```

Реакция на Ветвление:

```
f(x : Ω) = D(x)^b
```

Это **масштабно-ковариантные** функции. Они сохраняют свою форму при Акте. Показатель **b** — скорость роста относительно Хребта.

### Показательные (Экспоненциальные)

```
f(x) = D(x)
```

Реакция на Ветвление:

```
f(x : Ω) = D(D(x)) = D²(x)
```

Акт превращает удвоение входа в повторное ветвление выхода. Это закон ускоренного роста.

### Логарифмические (Навигационные)

```
L(x) — номер уровня Хребта
```

Реакция на Ветвление:

```
L(x : Ω) = L(x) ⊕ Id
```

Акт превращается в простой **сдвиг на уровень**. Логарифм измеряет глубину состояния на Хребте.

### Периодические (Циклические)

```
f(x) = e^{i·k·x}
```

Реакция на Ветвление:

```
f(x : Ω) = e^{i·k·D(x)}
```

Акт масштабирует фазу. Это вращение в Плоскости Потоков, описание волн и орбит.

## Композиция

```
(f ∘ g)(x) = f(g(x))
```

Применение законов последовательно. Ветвление проходит сквозь композицию:

```
(f ∘ g)(x : Ω) = f(g(D(x)))
```

## Парные Функции (Инверсия)

У каждого закона есть обратный, возвращающий к истоку:

| Закон `f(x)` | Возврат `f⁻¹(x)` | Смысл |
|---|---|---|
| `x^b` | `x^(Id : b)` (Корень) | Извлечение сути |
| `D(x)` | `L(x)` (Логарифм) | Измерение глубины |
| `D(x)` (Ветвление) | `x : D(Id)` (Сжатие) | Возврат к предку |

## Полный набор операций

| Операция | Степенная | Показательная | Логарифм | Периодическая |
|---|---|---|---|---|
| **: Ω** | `D(x)^b` | `D²(x)` | `L(x)+1` | `e^{ik·D(x)}` |
| **: D(Id)** | `(x:D(Id))^b` | `D(x:D(Id))` | `L(x)−1` | `e^{ik·(x:D(Id))}` |
| **⊕** | `x^b ⊕ y^b` | `D(x) ⊕ D(y)` | `L(x) ⊕ L(y)` | `e^{ikx} ⊕ e^{iky}` |
| **⊗** | `(x⊗y)^b` | `D(x⊗y)` | `L(x⊗y)` | `e^{ik(x⊗y)}` |

## Статус

Функции — не произвольные формулы. Это **инструменты навигации** по Носителю. Каждый класс отражает определённое отношение к Акту `Id : Ω`.

### II.2. Фабрики функций

Все фабрики возвращают `UFunction` — frozen dataclass с тремя обязательными полями:

| Поле | Тип | Семантика |
|------|-----|-----------|
| `apply` | `Callable[[object], object]` | `f(x)` — основное отображение |
| `branch_input` | `Callable[[object], object]` | `f(x : Ω)` — реакция на ветвление |
| `compress_input` | `Callable[[object], object]` | `f(x : D(Id))` — реакция на сжатие |

Публичные методы `UFunction`:

| Метод | Эквивалент | Описание |
|-------|-----------|----------|
| `__call__(x)` | `f(x)` | Вызов функции |
| `on_branch(x)` | `f(x : Ω)` | Ветвление входа |
| `on_compress(x)` | `f(x : D(Id))` | Сжатие входа |
| `compose(inner)` | `f ∘ inner` | Композиция (делегирование `compose()`) |

#### power_function(exponent)

- **Математика:** `f(x) = x^b`
- **apply:** `power(value, exponent)` → `src/core/algebra/power.py`
- **on_branch:** `power(branch(value), exponent)` → `D(x)^b`
- **on_compress:** `power(compress(value), exponent)` → `(x:D(Id))^b`
- **Параметр:** `exponent` — любой объект, приводимый к `float`
- **Масштабирование хребта:** `SpineLevel.scale(exponent)` → `D^(n·b)(Id)`

#### branching_function()

- **Математика:** `f(x) = D(x)`
- **apply:** `branch` → `src/core/algebra/branch.py`
- **on_branch:** `branch(branch(value))` → `D²(x)`
- **on_compress:** `branch(compress(value))` → `D(x:D(Id))`

#### logarithmic_function()

- **Математика:** `f(x) = L(x)` — номер уровня хребта
- **apply:** `L(value)` → `src/core/operators/L.py`
- **on_branch:** `L(value) + 1.0`
- **on_compress:** `L(value) - 1.0`
- **Сдвиг:** `+Id` / `-Id` на хребте

#### periodic_function(k)

- **Математика:** `f(x) = e^(i·k·x)` → `ComplexState`
- **apply:** `ComplexState(cos(k·x), sin(k·x))`
- **on_branch:** `apply(branch(value))` → `e^(i·k·D(x))`
- **on_compress:** `apply(compress(value))` → `e^(i·k·(x:D(Id)))`
- **Возврат:** `ComplexState(real, imaginary)` — не `float`
- **Зависимости:** `sin`, `cos` из `src/core/transcendental/`

## III. Зачем

### Инженерная аргументация

1. **Единый интерфейс для всех типов состояний.** `UFunction` — абстрактный адаптер, который принимает любой U-состояние (float, `SpineLevel`, `ComplexState`, `DualNumber`, `ParticipationRatio`, `FullnessShare`) и возвращает преобразованное состояние. Это позволяет писать алгоритмы навигации без знания конкретной числовой системы.

2. **Явное декларирование реакции на `branch` / `compress`.** Каждая фабрика функций задаёт три lambda-функции:
   - `apply` — основное отображение
   - `branch_input` — как функция ведёт себя, когда вход уже ветвится (`x : Ω`)
   - `compress_input` — как функция ведёт себя при сжатии (`x : D(Id)`)
   
   Это необходимо для корректной композиции: при `(f ∘ g)(x : Ω)` ветвление проходит через `inner` до `outer`, а не применяется к результату.

3. **Масштабная ковариантность степенных функций.** `power_function(b)` сохраняет свою форму при ветвлении: `f(x:Ω) = D(x)^b` — та же формула, но с ветвящимся аргументом. Это свойство критично для рекуррентных алгоритмов, где глубина рекурсии (`n` в `Dⁿ(Id)`) влияет на масштаб.

4. **Функция как инструмент навигации по хребту.** `logarithmic_function` — это `L()` оператор, который измеряет глубину состояния на хребте (`L(Dⁿ(Id)) = n`). Ветвление даёт `+1`, сжатие `−1`. Это позволяет алгебраически отслеживать глубину рекурсии без обхода структуры.

5. **Периодические функции как `ComplexState` а не `float`.** `periodic_function` возвращает `ComplexState(cos(θ), sin(θ))`, а не скаляр. Это необходимо для:
   - Компонентной композиции с `ComplexState.rotate()`
   - Масштабирования фазы через `branch()` / `compress()`
   - Нормы `|z|²` для анализа устойчивости

6. **Композиция как композиция ветвления.** `compose(f, g)` не просто `f(g(x))` — она также определяет `on_branch` и `on_compress` через проход ветвления через `inner` до `outer`:
   - `on_branch(x) = outer(inner.on_branch(x))`
   - `on_compress(x) = outer(inner.on_compress(x))`
   
   Это гарантирует, что реакция на `Ω` не теряется при композиции.

7. **Зависимости от трансцендентных функций.** Все четыре фабрики функций зависят от:
   - `power()` для степенных
   - `branch()` / `compress()` для всех
   - `sin()` / `cos()` для периодических
   - `L()` для логарифмических
   - `add()` / `multiply()` для промежуточных вычислений
