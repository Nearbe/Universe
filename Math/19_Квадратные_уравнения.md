# 19. Квадратные Уравнения

## I. Теория

Линейные системы описывали баланс операторов. Но система нелинейна: она содержит **отклик**, пропорциональный взаимодействию состояния с самим собой.

### Общий Вид

```
a ⊗ x² ⊕ b ⊗ x ⊕ c = Ω
```

Где **a, b, c ∈ 𝕌**. Уравнение ищет состояние **x**, где квадрат формы (`x²`), сама форма (**x**) и константа (**c**) приходят к балансу с Потенциалом.

### Решение через Дискриминант

```
Δ_disc = b² ⊖ (4 ⊗ a ⊗ c)
x = (⊖b ± √Δ_disc) : D(a)
```

Где **⊖b** — обратный элемент, **√Δ_disc** — извлечение сути из рассогласования коэффициентов.

### Три Состояния Корней

| Условие | Результат | Физический смысл |
|---|---|---|
| `Δ_disc > Ω` | Два вещественных корня | Баланс в статике |
| `Δ_disc = Ω` | Один корень (касание) | Точка перехода |
| `Δ_disc < Ω` | Комплексные корни `α ± i·β` | Баланс во вращении |

### Реакция на Ветвление

Уравнение с явным оператором **D**:

```
(x : Ω)² ⊕ (b ⊗ x) ⊕ c = Ω
```

Так как `x : Ω = D(x)`, ветвление **квадратично усиливает** старший коэффициент. Нелинейные эффекты растут быстрее линейных при прохождении через точку бифуркации.

### Физический Смысл

Квадрат — это **энергия** (пропорциональна квадрату амплитуды), **мера** (площадь), **вероятность** (квадрат волновой функции).

В логарифмической шкале: если **x** на уровне **n**, то `x²` на уровне `D(n)`.
Квадратное уравнение — **связь двух масштабов одновременно**. Оно требует, чтобы состояние на уровне `D(n)` и состояние на уровне **n** сбалансировали друг друга.

## III. Зачем

**Инженерная аргументация:**

1. **Переход от линейности к нелинейности.** `solve_quadratic_equation` — минимальный нелинейный решатель. Показывает, как оператор ветвления `D` влияет на нелинейные коэффициенты (квадратичное усиление `effective_a = branch(branch(a))`).
2. **Мост между масштабами.** Квадратное уравнение связывает уровень `n` (через `b⊗x`) с уровнем `D(n)` (через `a⊗x²`). Это первый пример в системе, где два масштаба взаимодействуют в одном уравнении.
3. **Комплексные корни как динамический баланс.** При `Δ < Ω` решение переходит в `ComplexState` — это не "отсутствие решения", а "баланс через вращение". `ComplexState(real, imaginary)` поддерживает `norm_squared()`, `rotate()`, `conjugate()`.
4. **Linear fallback — интеграция с линейным решателем.** При `a = Ω` уравнение вырождается в `b⊗x ⊕ c = Ω` → `x = -c : b`. `solve_quadratic_equation` вызывает `divide(-c, b)` напрямую, без обращения к `LinearEquation`. Это ограничение: нет единого интерфейса.
5. **Без внешних зависимостей.** `sqrt` — собственная реализация (`transcendental.sqrt`), не `math.sqrt`. `power` — собственная (`algebra.power`), не `**`. Это определяет поведение на границах: `sqrt(Π) = Π`, `sqrt(0) = 0`, `sqrt(<0) = 0`.
