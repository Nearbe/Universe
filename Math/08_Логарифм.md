# 08. Логарифм

## I. Теория

### 1. Определение

Логарифм в системе U — это **шкала глубины ветвления**, а не внешняя аналитическая функция. Он отвечает на вопрос: *сколько тактов оператора ветвления `D` потребовалось для достижения состояния `a` от `Id`*.

$$L(a) = \log_2 |a|$$

где основание логарифма — собственное значение класса ветвления $D(\text{Id}) = 2$.

### 2. Логарифмическая ось

Логарифм вводит на носителе **одномерную координату** — ось глубины. На этой оси:

- Умножение (вертикальная глубина, $\otimes$) превращается в сложение (горизонтальная длина, $\oplus$):

$$L(a \otimes b) = L(a) \oplus L(b)$$

- Ветвление — сдвиг $+1$:

$$L(a : \Omega) = L(a) + 1$$

- Сжатие (спуск) — сдвиг $-1$:

$$L(a : D(\text{Id})) = L(a) - 1$$

### 3. Закон противоположностей

Система зеркально сбалансирована относительно центра $\text{Id}$ (глубина $0$):

$$L(a : D(\text{Id})) = -L(a)$$

Обратная величина — не «мало». Это отражение в другую сторону по той же оси.

### 4. Алгебраические границы

| Операция | Сдвиг | Смысл |
|---|---|---|
| $a : \Omega$ (Ветвление) | $+1$ | Подъём на уровень хребта |
| $a : D(\text{Id})$ (Сжатие) | $-1$ | Возврат к истоку |
| $\Pi$ | $+\infty$ | Абсолютная сложность |
| $\Omega$ | $-\infty$ | Абсолютный потенциал |

### 5. Полный набор операций

| Операция | Результат |
|---|---|
| $L(a : \Omega)$ | $L(a) + 1$ |
| $L(a : D(\text{Id}))$ | $L(a) - 1$ |
| $L(a^b)$ | $b \cdot L(a)$ |
| $L(\sqrt[n]{a})$ | $L(a) : n$ |
| $L(\text{Id})$ | $0$ |
| $L(\Pi)$ | $+\infty$ |
| $L(\Omega)$ | $-\infty$ |

### 6. Скалярная и дуальная матрицы ветвления

В базисе $\{\text{Id}\}$:

$$[D(\text{Id})] = \begin{pmatrix} 2 \end{pmatrix}$$

В базисе $\{\text{Id}, \varepsilon\}$ (дуальные числа):

$$[D(\text{Id})] = \begin{pmatrix} 2 & 0 \\ 0 & 2 \end{pmatrix}$$

Ядро: $\operatorname{Ker} D = \{\Omega\}$. Образ: $\operatorname{Im} D = \mathbb{U}$.

### II.3. Что реализовано

- **[✓]** `L(x)` — полиморфный dispatch: SpineLevel → depth, Π → `+∞`, Ω → `−∞`, scalar → `log2(|x|)`, vector → `map_scalar_or_vector`
- **[✓]** `log2(x)` — базовая реализация через `math.log2` с алгебраическими границами $\Omega$ и $\Pi$
- **[✓]** `ln(x)` — натуральная логарифмическая шкала через разложение по $e$ и ряд Тейлора $\ln(x) = 2\sum \frac{y^{2k+1}}{2k+1}$, $y = \frac{x-1}{x+1}$
- **[✓]** `LogInfinity` — типизированные алгебраические бесконечности с операциями $+$, $\times$, $-$
- **[✓]** `depth_add()` — сложение глубин; замкнутые бесконечности одного знака; ошибка при сложении противоположных
- **[✓]** `depth_scale()` — умножение глубины; $\alpha < 0$ инвертирует знак бесконечности; $\alpha = 0 \to 0$
- **[✓]** `SpineLevel.level()` — прямой доступ к глубине $n$
- **[✓]** `SpineLevel.scale()` / `root()` — $L(a^b)$ и $L(\sqrt[n]{a})$
- **[✓]** `SpineLevel.branch()` / `compress()` — сдвиги $\pm 1$
- **[✓]** `BranchingOperator` — ядро $\{\Omega\}$, образ $\mathbb{U}$, скалярная и дуальная матрицы
- **[✓]** `Fullness` (Π) — алгебраическая бесконечность ветвления

## III. Зачем

Логарифм — **единственный инструмент навигации по носителю**. Без него:

- Нет метрики расстояния между состояниями: ветвление $a : \Omega$ — это «ещё одно состояние», но нельзя измерить *насколько далеко* от $\text{Id}$.
- Умножение $\otimes$ остаётся непрозрачным: невозможно перевести вертикальную глубину в горизонтальную длину.
- Система не может выполнять **логарифмические сдвиги** — переход между масштабами рекурсии.
- `SpineLevel` теряет свой смысл: глубина $n$ — это и есть $L(D^n(\text{Id}))$, но без `L` нет обобщения на нехребтовые состояния.

Логарифм — это **внутренний счётчик тактов** системы, а не внешняя функция. Каждая операция ветвления/сжатия автоматически сдвигает систему по логарифмической оси.
