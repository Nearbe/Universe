# 11. Рождение Конечного

## I. Теория

Полнота (Π) — база системы. Из Полноты рождается Конечное через отношение долей.

### Деление Полноты

`Π : D(Id)` — снижение интенсивности (не уменьшение «размера»). Полнота остаётся фоном, доля — то, сколько от неё активно.

### Возврат к Единству

Два механизма генерации конечного из бесконечного:

1. **Отношение равных долей:** `(Π : D(Id)) : (Π : D(Id)) = Id`
2. **Тождество Полноты самой себе:** `Π : Π = Id`

Оба сходятся: **Id — не величина, а отношение Полноты к себе.**

### Такты Порождения

| Такт | Процесс | Результат |
|---|---|---|
| **Id** | Исходное состояние | Единство |
| **Id : Ω** | Ветвление | `D(Id)` — Дуальность |
| **Π : D(Id)** | Сжатие Полноты | Доля |
| **(Π : D(Id)) : (Π : D(Id))** | Отношение долей | `Id` — Конечное |

### II.2. Функции/классы

**divide(numerator, denominator) — src/core/algebra/divide.py**

Правила Π-ветви (в порядке проверки):

| Условие | Результат | Строка в коде |
|---|---|---|
| `is_fullness(numerator) and is_fullness(denominator)` | `IDENTITY` (1.0) | `divide` Π-блок |
| `is_fullness(numerator) and not is_fullness(denominator)` | `PI` | `divide` Π-блок |
| `is_fullness(denominator) and not is_fullness(numerator)` | `OMEGA` | `divide` Π-блок |

FullnessShare.divide_by(other):
- `isinstance(other, FullnessShare)` → `divide(self.intensity, other.intensity)`
- `other == PI` → `FullnessShare(self.intensity)` (без изменений)
- Иначе → `FullnessShare(divide(self.intensity, other))`

ParticipationRatio.divide_by(denominator):
- `is_omega(denominator)` → `self.branch()`
- Иначе → `ParticipationRatio(numerator, denom * denominator)`

**power(base, exponent) — src/core/algebra/power.py**

Правила Ω/Π:

| Условие | Результат |
|---|---|
| `is_fullness(exponent)` (любой base, не Ω) | `PI` |
| `is_omega(exponent) and is_omega(base)` | `OMEGA` |
| `is_omega(exponent)` | `IDENTITY` |
| `is_fullness(base)` | `PI` |

**multiply(left, right) — src/core/algebra/multiply.py**

Π-правила:
- `is_fullness(left) or is_fullness(right)` → `PI`

**add(left, right) — src/core/algebra/add.py**

Π-правила:
- `is_fullness(left) or is_fullness(right)` → `PI`

**branch(value) — src/core/algebra/branch.py**

Π-замкнутость:
- `is_fullness(value)` → `PI`

**compress(value) — src/core/algebra/compress.py**

Π-замкнутость:

### Что сломится при изменении

1. **Изменение `Π : Π` с `IDENTITY` на другое значение** — сломит все выражения типа `(Π:a):(Π:a)` (возврат к Единству). 12+ тестов в `test_u_algebra.py` и `test_fullness_share.py`.

2. **Изменение `PI` на `None`/`float('inf')`** — `is_fullness()` перестанет работать, `divide()` упадёт на `TypeError` или вернёт неверный результат.

3. **Изменение `IDENTITY` с `1.0`** — `(Π:4):(Π:4)` вернёт неверный результат. `power(5, Ω)` → `IDENTITY` также сломается.

4. **Удаление `FullnessShare`** — потеря механизма `(a:Π):(b:Π) = a:b`. `test_fullness_share.py` не запустится.

5. **Изменение `D_ID` с `2.0`** — `compress()` делит на `D_ID`, `branch()` умножает на `D_ID`. Все spine-уровни, `%`-алгебра, логарифмическая шкала.
