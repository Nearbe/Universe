# 12. Алгебра Долей

## I. Теория

Множество долей **ℙ** — поле отношений участия в Полноте (Π). Каждая доля `a : Π` — не «число», а **уровень вовлечённости** в Акт.

```
ℙ = { a : b | a, b ∈ 𝕌, b ∉ Ω }
```

Полнота **Π** — предел ветвления:
```
Π = lim Dⁿ(Id),    n → ∞
```

Доли подчиняются тем же операциям, что и 𝕌, но с фиксированным знаменателем Π:

| Операция | Формула | Смысл |
|---|---|---|
| Сложение | `(a : Π) ⊕ (b : Π) = (a ⊕ b) : Π` | Сумма вкладов |
| Умножение | `(a : Π) ⊗ (b : Π) = (a ⊗ b) : Π` | Пересечение долей |
| Деление | `(a : Π) : (b : Π) = a : b` | Сравнение интенсивностей |
| Ветвление | `(a : Π) : Ω = D(a) : Π` | Удвоение участия |
| Сжатие | `(a : Π) : D(Id) = (a : D(Id)) : Π` | Ослабление связи |

**Тождество Состояний** (онтологическое, не числовое):
```
Id ≡ D(Id) ≡ Π ≡ Π : Π
```

**Инвариантность Π**:
```
√[n]{Π} = Π,    Π^k = Π,    Π : Ω = Π,    Π : D(Id) = Π
```

## III. Зачем

Алгебра долей — **инженерный слой** между абстрактной теорией множеств (00_Теория_множеств.md) и прикладной математикой.

1. **Типобезопасные операции надΠ** — `FullnessShare`, `ParticipationRatio`, `BranchProbability` инкапсулируют доли как типы, а не float. Это предотвращает смешение абсолютных и относительных координат.

2. **Единый диспатчер ветвления/сжатия** — `branch()`, `compress()`, `divide()`, `multiply()`, `add()`, `power()` — один интерфейс для 6 типов состояний + скаляры. Добавление нового типа состояния требует изменения в 6 файлах (`_fullness_share`, `_participation_ratio`, `_spine_level`, `_dual_number`, `_complex_state`, `_complex_fullness` диспетчеры).

3. **Процентная алгебра как логарифмическое сложение** — `percentage_multiply(a, b) = 2^((ridge(a)+ridge(b))/2)` — позволяет работать с процентами в логарифмическом пространстве, где сложение заменяется умножением.

4. **Вероятность как комбинаторная доля** — `BranchProbability` связывает комбинаторику (число ветвей) с алгеброй долей (отношение). `binomial_probability(k, n)` вычисляет `C(n,k) : Dⁿ(Id)`.

5. **Исчисление на ветвящейся структуре** — `branched_derivative`, `branched_integral`, `compressed_derivative`, `compressed_integral` — производные и интегралы для функций от ветвящихся/сжатых аргументов.

6. **Свеп для анализа распределений** — `compute_sweep` генерирует пороги, вычисляет occupancy rates, обнаруживает jump events. `rg_aware_sweep` — RG-осознанные пороги.
