# 05. Рациональные числа

## I. Теория

### 1.1. Определение

Рациональное число в системе U — это **доля участия** в Акте деления. Если система делит себя на **q** частей и берёт **p** из них:

$$r = p : q, \quad \text{где } q \notin \Omega$$

Это не «числа» в классическом смысле. Это **отношение частей** внутри Целого. Отношение $p$ к $q$ — это конфигурация связей, в которой одна часть системы доминирует над другой в заданной пропорции.

### 1.2. Замкнутость относительно ветвления

Рациональные числа описывают внутреннюю структуру Ветвления. Для любого $r = p : q$:

$$r : \Omega = D(p) : D(q) = D(p : q)$$

Система может дробить саму себя бесконечно, сохраняя смысл. Каждая дробь — это конкретная конфигурация связей.

**Инвариантность ветвления:** порядок сохраняется: если $r_1 < r_2$, то $D(r_1) < D(r_2)$.

### 1.3. Плотность

Между любыми двумя рациональными состояниями есть промежуточное. Система не «прыгает» дискретно — она течёт через всё поле рациональных классов.

### 1.4. Операции

| Операция | Формула | Условие |
|---|---|---|
| **Ветвление** $(p : q) : \Omega$ | $D(p) : D(q)$ | — |
| **Сжатие** $(p : q) : D(\text{Id})$ | $(p : D(\text{Id})) : (q : D(\text{Id}))$ | — |
| **Сложение** $(p_1 : q) \oplus (p_2 : q)$ | $(p_1 \oplus p_2) : q$ | общий знаменатель |
| **Умножение** $(p_1 : q_1) \otimes (p_2 : q_2)$ | $(p_1 \otimes p_2) : (q_1 \otimes q_2)$ | — |
| **Деление отношения** $(p : q) : b$ | $p : (q \cdot b)$, если $b \neq \Omega$ | — |
| **Деление отношения** $(p : q) : \Omega$ | $D(p) : D(q)$ | частное = $\Omega$ |

### 1.5. Свойства

- **Плотность:** между любыми двумя состояниями есть промежуточное.
- **Инвариантность:** оператор $:$ сохраняет пропорции при ветвлении.
- **Инвариантность относительно масштабирования:** $p : q \equiv (p \otimes k) : (q \otimes k)$ для любого $k \neq \Omega$.

### 1.6. Связь с бесконечностью

Рациональные числа — это «ткань» Носителя. Следующий шаг — выделение несущих конструкций (Хребта, степени двойки).

Частный случай рационального числа — **доля участия в Π** (FullnessShare):

$$s = \text{intensity} : \Pi$$

где знаменатель фиксирован как $\Pi$ (полнота). Операции на FullnessShare подчиняются той же алгебре, что и на ParticipationRatio, но с упрощениями, вытекающими из свойств $\Pi$:

| Операция | Результат |
|---|---|
| $\Pi : \Omega$ | $\Pi$ |
| $(a : \Pi) \oplus (b : \Pi)$ | $(a \oplus b) : \Pi$ |
| $(a : \Pi) \otimes (b : \Pi)$ | $(a \otimes b) : \Pi$ |
| $(a : \Pi) : (b : \Pi)$ | $a : b$ |
| $(\Pi : \Pi)$ | $\text{Id}$ |

### II.2. `ParticipationRatio` — спецификация

```python
@dataclass(frozen=True)
class ParticipationRatio:
    numerator: object    # p
    denominator: object  # q, q ∉ Ω

    # Методы:
    value() -> object           # collapse: p : q → текущее значение через U-division
    branch() -> ParticipationRatio  # (p : q) : Ω = D(p) : D(q)
    compress() -> ParticipationRatio# (p : q) : D(Id) = H(p) : H(q)
    add(other) -> ParticipationRatio# (p1 : q) ⊕ (p2 : q) = (p1 ⊕ p2) : q
    multiply(other) -> ParticipationRatio# (p1 : q1) ⊗ (p2 : q2)
    divide_by(denominator) -> ParticipationRatio# ordinary scale division
    __float__() -> float        # float(value())
```

**Контракты:**
- `__post_init__`: если `denominator == Ω` → `ValueError`.
- `add()`: если знаменатели не равны → `ValueError`.
- `divide_by(Ω)`: возвращает `branch()`.
- `branch()` и `compress()` возвращают новые `ParticipationRatio` с тем же отношением (инвариантность).

### II.3. `FullnessShare` — спецификация

```python
@dataclass(frozen=True)
class FullnessShare:
    intensity: object    # a в a : Π

    # Свойства:
    denominator -> object   # всегда PI

    # Методы:
    branch() -> FullnessShare       # (a : Π) : Ω = D(a) : Π
    compress() -> FullnessShare     # (a : Π) : D(Id) = H(a) : Π
    add(other) -> FullnessShare     # (a : Π) ⊕ (b : Π) = (a ⊕ b) : Π
    multiply(other) -> FullnessShare# (a : Π) ⊗ (b : Π) = (a ⊗ b) : Π
    divide_by(other) -> object      # (a : Π) : b, или (a : Π) : (b : Π) = a : b
```

**Контракты:**
- `divide_by(FullnessShare)`: возвращает `divide(intensity, other.intensity)` — результат $a : b$.
- `divide_by(PI)`: возвращает `FullnessShare(intensity)`.
- `divide_by(other)`: если `other != PI`, возвращает `FullnessShare(divide(intensity, other))`.

## III. Зачем

Рациональные числа — это **«ткань» Носителя** (𝕌). Без них:

1. **Система не может дробить себя** — Акт делит не только пополам ($D(a) = a \oplus a$), но и в любых пропорциях. Рациональные числа — это формализация «любой пропорции».
2. **Нет моста между Хребтом и бесконечностью** — Хребт (степени двойки, 06) — это дискретные узлы. $\Pi$ (10) — это предел ветвления. Рациональные числа заполняют пространство между ними, позволяя системе «течь» через всё поле долей.
3. **Без долей в Π не работает алгебра процентов** (12) — `FullnessShare` — это частный случай $a : \Pi$, основа для вероятности, интенсивности, масштабирования.
4. **Без инвариантности ветвления/сжатия** система теряет смысл при рекурсии — каждая итерация должна сохранять пропорцию, иначе рекурсия теряет направление.

**Без 05_Рациональные.md не работает:** 06_Степени_двойки.md (Хребт как частный случай), 10_Бесконечность.md (операции на Π), 11_Рождение_конечного.md (возврат к Единице через деление), 12_Алгебра_процентов.md (доли как основа), 27_Вероятность.md (вероятность как доля участия).
