# 24. p-адические Числа (D(Id)-адическая топология)

## I. Теория

### I.1. Неархимедова метрика

В стандартной топологии ℝ метрика индуцируется абсолютным значением `|x|`. В p-адической топологии метрика индуцируется **p-адической нормой**:

```
|x|_p = p^(-v_p(x))
```

где `v_p(x)` — максимальная степень `p`, на которую `x` делится без остатка (p-адическая оценка).

Для данной системы `p = D(Id) = 2`, поэтому это **2-адическая топология**.

### I.2. Две топологии системы

| Свойство | ℝ (реальность) | ℚ₂ (глубина) |
|---|---|---|
| Направление `: Ω` | Рост к Π (экспансия) | Сжатие к Ω (фокус) |
| Бесконечность | Предел удаления | Предел углубления |
| Смысл | «Насколько много?» | «Насколько точно?» |
| Метрика | `|a - b|` | `2^(-v₂(a-b))` |
| Ультраметрика | ❌ | ✅ `d(x,z) ≤ max(d(x,y), d(y,z))` |

### I.3. D(Id)-адические целые

D(Id)-адические целые числа — это последовательности остатков, согласованных по уровням точности:

```
ℤ_D = lim← (𝕌 : Dⁿ(Id))
```

Каждый остаток — снимок состояния на уровне точности `Dⁿ(Id) = 2ⁿ`.

### I.4. Соленоид и p-адические числа

Соленоид — геометрическая реализация D(Id)-адических чисел. Это одно и то же пространство, увиденное с разных сторон.

### I.5. Предел различимости

В D(Id)-адической топологии существует предел, после которого деление пополам перестаёт менять состояние. Это свойство дискретности системы — минимальный «пиксель» различимости.

### II.2. Функции/классы

#### `v2_adic_valuation(val: float | int) → float | LOG_POSITIVE_INFINITY`

**Модуль:** `src/core/number_theory/v2_adic_valuation.py`

**Назначение:** Вычисление 2-адической оценки `v₂(val)` — максимальная степень 2, на которую `val` делится.

**Поведение:**
- `v₂(0) = LOG_POSITIVE_INFINITY` (логарифмическая ось бесконечности)
- `v₂(2ᵏ·m) = k` для нечётного `m` и `m ≠ 0`
- Для нецелых float: coercing к ближайшему целому (`_coerce_integer_delta`)
  - `value == 0.0` → возврат `0` (не бесконечность, так как это float-ноль, не математический ноль)
  - `round(value) == 0` (но `value ≠ 0`) → возврат `1` (защита от деления на ноль)
- Отрицательные числа: берётся `abs(integer)` перед факторизацией

**Сложность:** O(log |n|) — итеративное деление на 2.

**Зависимости:** `LOG_POSITIVE_INFINITY` из `src/core/utils/logarithmic_axis.py`.

---

#### `p_adic_distance(a, b) → float | list[float]`

**Модуль:** `src/core/metrics/p_adic_distance.py`

**Назначение:** Вычисление 2-адического расстояния между двумя значениями или векторами.

**Поведение:**
- Скалярный случай: `d₂(a, b) = 2^(-v₂(a-b))`
  - `a == b` → `0.0` (через `v₂(0) = +∞` → `2^(-∞) = 0`)
- Векторный случай: поэлементное вычисление, возвращает `list[float]`
  - `is_vector()` / `zip_vectors()` для обнаружения и итерации
- Non-integer floats: coercing к целому через `v2_adic_valuation`

**Зависимости:** `v2_adic_valuation`, `LOG_POSITIVE_INFINITY`, `is_vector`, `zip_vectors` из `src/core/utils/vectorization.py`.

---

#### `SpineLevel`

**Модуль:** `src/core/states/spine.py`

**Назначение:** Представление уровня хребта `Dⁿ(Id) = 2ⁿ`.

**Поля:** `depth: float` (≥ 0, шаг `DEPTH_STEP = 1.0`)

**Методы:**
| Метод | Формула | Результат |
|---|---|---|
| `branch()` | `n → n+1` | `Dⁿ⁺¹(Id)` |
| `compress()` | `n → n-1` | `Dⁿ⁻¹(Id)` (≥ 0) |
| `scale(exp)` | `n → n·exp` | `D^(n·exp)(Id)` |
| `root(deg)` | `n → n/de g` | `D^(n/deg)(Id)` |
| `value()` | `2ⁿ` | `float` |
| `level()` | `n` | `float` |

**Константы:** `MIN_SPINE_DEPTH = 0.0`, `DEPTH_STEP = 1.0`, `D_ID = 2.0`.

**Связь с p-adic:** `SpineLevel` определяет масштаб точности. Уровень `n` соответствует модулю `2ⁿ` в p-адической разрядной записи.

---

#### `DualNumber`

**Модуль:** `src/core/states/dual_number.py`

**Назначение:** Дуальное состояние `x + v·ε` с `ε² = Ω`.

**Связь с p-adic:** `DualNumber` используется как тип в `divide()` и `branch()`/`compress()` для обработки дуальных компонентов при p-адических вычислениях. `ε² = Ω` означает, что дуальная часть коллапсирует в потенциал.

---

#### `ComplexState`

**Модуль:** `src/core/states/complex_plane.py`

**Назначение:** Комплексное состояние `x + i·y` с `i² = -Id`.

**Связь с p-adic:** Используется в `divide()` для p-адических делений комплексных чисел. `divide_by()` обрабатывает случай `norm == OMEGA` как ветвление.

---

#### `ParticipationRatio`

**Модуль:** `src/core/states/rational.py`

**Назначение:** Соотношение участия `numerator : denominator`.

**Связь с p-adic:** `branch()` и `compress()` действуют на обе части. `divide_by()` при `denominator == Ω` вызывает ветвление.

---

#### `FullnessShare`

**Модуль:** `src/core/states/fullness_share.py`

**Назначение:** Доля в Π: `intensity : Π`.

**Связь с p-adic:** `divide_by()` при `other == Π` возвращает `divide(intensity, Π)` — p-адическая редукция к конечному значению.

#### Алгебраические операции

| Операция | Модуль | Поведение при p-adic |
|---|---|---|
| `branch(value)` | `src/core/algebra/branch.py` | `D(value) = value : Ω`. Dispatch по типам: DualNumber → share → complex → spine → ratio → scalar. Для float: `scalar * D_ID` |
| `compress(value)` | `src/core/algebra/compress.py` | `H(value) = value : D(Id)`. Аналогичный dispatch. Для float: `scalar / D_ID` |
| `divide(a, b)` | `src/core/algebra/divide.py` | `a : b`. Special cases: `a : Ω = D(a)`, `Ω : b = Ω`, `Π : Π = Id`. Для float: `a / b` |
| `power(base, exp)` | `src/core/algebra/power.py` | `base^exp`. Special: `Ω^Ω = Ω`, `Π^Ω = PI`, `Π^Π = Id`. Для spine: `level.scale(exp)` |
| `multiply(a, b)` | `src/core/algebra/multiply.py` | `a ⊗ b`. Special: `Ω` аннигилирует форму, `Π` замыкает. Для float: `a * b` |

#### `Fullness` / `PI`

**Модуль:** `src/core/infinity/infinity.py`

**Назначение:** Алгебраическое состояние Π (полнота/бесконечность).

**Связь с p-adic:** `PI` — фиксированный элемент (`Final[Fullness]`). В p-адической топологии `Π` является пределом ветвления, но не пределом p-адической сходимости (p-адические числа могут сходиться к конечным значениям).

---

#### `scale_invariance`

**Модуль:** `src/core/renormalization/scale_invariance.py`

**Назначение:** Ренормализационная группа (RG) — масштабная инвариантность.

**Ключевые функции:**
| Функция | Формула | Описание |
|---|---|---|
| `rg_step(value)` | `branch(value)` | RG шаг = ветвление |
| `inverse_rg_step(value, scale=D_ID)` | `value : scale` | Обратный RG шаг |
| `rg_flow(seed, steps)` | итеративное ветвление | Последовательность ренормализованных состояний |
| `scale_dimension(value)` | `L(value)` | Скалярное измерение |
| `beta_function(value)` | scaling rate | Функция бета |
| `fixed_point_holds(value, op)` | `|op(v) - v| < ε` | Проверка неподвижной точки |

**Связь с p-adic:** RG-шаг с масштабом `D_ID = 2` — это переход между p-адическими уровнями. `scale_dimension` возвращает `L(value) = log₂(value)`, что является p-адической оценкой.

#### `branched_derivative` / `compressed_derivative`

**Модуль:** `src/core/calculus/branched_derivative.py` / `src/core/calculus/compressed_derivative.py`

**Назначение:** Производные через ветвление и сжатие.

| Функция | Формула | Описание |
|---|---|---|
| `branched_derivative(fn, point)` | `D·f'(D(x))` | Производная g(x) = f(D(x)) |
| `compressed_derivative(fn, point)` | `f'(H(x)) : D` | Производная h(x) = f(x:D) |

**Связь с p-adic:** Эти производные — дискретные аналоги p-адических производных. В p-адическом анализе производная определяется через ультраметрическую норму.

**Связь с p-adic:** `depth` соответствует p-адической глубине. `address(value)` возвращает двоичный путь (p-адическая разрядная запись с основанием 2).

## III. Зачем

### III.1. Инженерная аргументация

**1. Двойная топология системы.** Система U существует одновременно в ℝ и ℚ₂. ℝ даёт «пространство» (размах от источника), ℚ₂ даёт «структуру» (точность на уровне). Это необходимо для:
- Анализа сходимости в p-адической метрике (неархимедова ультраметрика)
- Разделения экспансии (ℝ) и фокусировки (ℚ₂) в алгоритмах
- Определения предела различимости — минимального масштаба, на котором система ещё различает состояния

**2. p-адическая оценка как измеритель глубины.** `v₂(n)` — это не просто числовая функция. Это **измеритель глубины ветвления**:
- `v₂(n) = k` означает, что `n` делится на `2ᵏ` — то есть прошло через `k` уровней ветвления
- `SpineLevel.depth` = `v₂` для соответствующих значений
- `scale_dimension(value) = L(value)` — логарифмическая шкала, эквивалентная p-адической оценке

**3. RG-шаг = переход между p-адическими уровнями.** `rg_step(value) = branch(value)` масштабирует на `D_ID = 2`. Это означает, что каждый RG-шаг — это переход между p-адическими уровнями точности. `inverse_rg_step` — обратный переход.

**4. Ультраметрика для иерархической кластеризации.** Ультраметрическое неравенство `d(x,z) ≤ max(d(x,y), d(y,z))` означает, что все точки в p-адической метрике образуют иерархическую структуру (дерево). Это используется в:
- `rg_aware_sweep` — иерархический sweep с уровнями точности `2^(-level)`
- `BranchingLevel.address` — двоичный адрес = p-адическая разрядная запись

**5. Дискретность как фундаментальное свойство.** Предел различимости в p-адической топологии — это не «физическая константа», а свойство самой системы: минимальный шаг `2⁻ⁿ`, после которого `d(x, x + 2⁻ⁿ) = 1`, и система перестает различать.
