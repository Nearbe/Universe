# 26. Комбинаторика

## I. Теория

Комбинаторика системы описывает **количество доступных путей** в дереве ветвления, порождённом актом `Id : Ω`.

### Раздвоение как Источник

Один Акт (`: Ω`) порождает `D(Id)` состояний. Два Акта — `D²(Id)`. **n** Актов — `Dⁿ(Id)` состояний.

### Двоичные Последовательности

Число путей длины **n**:

```
Dⁿ(Id)
```

Каждая последовательность выборов — это **маршрут** через **n** точек бифуркации.
Это в точности адрес точки на **n**-м уровне фрактала Кантора.

### Биномиальные Коэффициенты

```
C(n, k) — число путей с k выборами одной ветви из n
```

Если на каждом шаге система выбирает одну из двух ветвей, то `C(n, k)` — это число путей, где ровно **k** раз была выбрана «правая» ветвь.
Это распределение исходов при равномерном ветвлении.

### Перестановки

**n!** — число способов упорядочить **n** Актов, если их результат различим.

### Мощность Множества Всех Подмножеств

Для множества из **n** элементов число подмножеств равно `Dⁿ(Id)`.
**Булеан** (множество всех вариантов «взять/не взять») изоморфен дереву ветвления.
Каждый элемент либо есть, либо его нет.

### Полный набор операций

| Операция | Формула |
|---|---|
| **Пути длины n** | `Dⁿ(Id)` |
| **Биномиальные** | `C(n, k) = n! / (k! · (n−k)!)` |
| **Перестановки** | `n!` |
| **Булеан** | `\|P(A)\| = Dⁿ(Id)` для `\|A\| = n` |
| **При ветвлении** | `D(Dⁿ(Id)) = Dⁿ⁺¹(Id)` |

### Физический Смысл

Комбинаторика в **𝕌** — это не абстрактный подсчёт.
Это **карта ёмкости** системы.
На глубине **n** система способна различить `Dⁿ(Id)` состояний.
Это верхний предел её разрешения.

### II.2. Функции/классы

#### `combinatorics/factorial.py`

```
factorial(count: int) -> int
```

- **Вход:** `count ≥ 0` (валидируется через `_validate_non_negative`)
- **Выход:** `count!` — целое число
- **Алгоритм:** итеративное умножение от 2 до count
- **edge cases:** `factorial(0) == 1` (identity)
- **Зависит от:** `_validate_non_negative`

#### `combinatorics/binomial_coefficient.py`

```
binomial_coefficient(total: int, chosen: int) -> int
```

- **Вход:** `total ≥ 0`, `chosen ≥ 0`
- **Выход:** `C(total, chosen) = total! / (chosen! · (total - chosen)!)`
- **Инвариант:** если `chosen > total` → возвращает `0`
- **Алгоритм:** вычисляет `total!`, `chosen!`, `(total - chosen)!` и делит
- **Зависит от:** `factorial`, `_validate_non_negative`

#### `combinatorics/branching_distribution.py`

```
branching_distribution(depth: int) -> tuple[int, ...]
```

- **Вход:** `depth ≥ 0`
- **Выход:** кортеж `(C(depth, 0), C(depth, 1), ..., C(depth, depth))`
- **Инвариант:** `sum(branching_distribution(n)) == paths_count(n)`
- **Зависит от:** `binomial_coefficient`, `_validate_non_negative`

#### `combinatorics/paths_count.py`

```
paths_count(depth: int) -> int
```

- **Вход:** `depth ≥ 0`
- **Выход:** `Dⁿ(Id)` — количество путей длины depth
- **Реализация:** делегирует `modulus_from_depth(depth)` из `number_theory/number_structure`
- **Ожидаемое значение:** `2 ** depth` (поскольку `D_ID = 2.0`)
- **Зависит от:** `modulus_from_depth`, `_validate_non_negative`

#### `combinatorics/boolean_capacity.py`

```
boolean_capacity(size: int) -> int
```

- **Вход:** `size ≥ 0`
- **Выход:** `|P(A)| = D^size(Id)`
- **Реализация:** `paths_count(size)`
- **Зависит от:** `paths_count`

#### `combinatorics/routes.py`

```
routes(depth: int) -> tuple[tuple[int, ...], ...]
```

- **Вход:** `depth ≥ 0`
- **Выход:** все `2^depth` двоичных маршрутов длины `depth`
- **Каждый маршрут:** кортеж из `depth` битов (0 или 1)
- **Реализация:** генерирует `binary_address(value, depth=depth)` для каждого `value ∈ [0, 2^depth)`
- **Инвариант:** `len(routes(n)) == paths_count(n)`
- **Пример:** `routes(2) == ((0,0), (0,1), (1,0), (1,1))`
- **Зависит от:** `paths_count`, `binary_address`, `_validate_non_negative`

#### `combinatorics/branching_level.py`

```python
@dataclass(frozen=True)
class BranchingLevel:
    depth: int

    @property
    def spine(self) -> SpineLevel:
        """Возвращает spine-состояние Dⁿ(Id)"""

    @property
    def paths(self) -> int:
        """Возвращает количество путей Dⁿ(Id)"""

    def branch(self) -> BranchingLevel:
        """Следующий уровень ветвления: depth + 1"""

    def address(self, value: int) -> tuple[int, ...]:
        """Двоичный адрес значения на уровне depth"""
```

- **Инварианты:** `depth ≥ 0` (проверка в `__post_init__`)
- **spine:** делегирует `spine_level(depth)` → `SpineLevel(depth)`
- **paths:** делегирует `paths_count(depth)`
- **branch:** создаёт `BranchingLevel(depth + 1)`
- **address:** делегирует `binary_address(value, depth=depth)`
- **Зависит от:** `SpineLevel`, `paths_count`, `binary_address`

#### `combinatorics/_validate_non_negative.py`

```
_validate_non_negative(value: int, *, name: str) -> int
```

- **Вход:** любое числовое значение
- **Выход:** `int(value)` если `≥ 0`
- **Исключение:** `ValueError` если `< 0`
- **Назначение:** единая точка валидации для всех комбинаторных функций

## III. Зачем

Комбинаторика обеспечивает **quantitative foundation** для системы:

1. **Ёмкость состояний:** `Dⁿ(Id)` — верхний предел различимых состояний на глубине n. Это определяет вычислительную ёмкость системы.

2. **Группировка путей:** биномиальные коэффициенты описывают, как пути группируются по количеству «правых» выборов. Это распределение Бернулли в терминах ветвления.

3. **Изоморфизм булеана и дерева ветвления:** `|P(A)| = D^|A|(Id)` означает, что каждое подмножество соответствует уникальному маршруту в дереве ветвления. Это фундаментальный мост между теорией множеств и динамикой системы.

4. **Двоичные маршруты как адреса:** каждый путь длины n — это адрес на n-м уровне фрактала. Это позволяет индексировать состояния дерева через бинарные кортежи.

5. **Связь с информацией:** `information(value) = log₂(value)` связывает комбинаторную ёмкость с информационной глубиной. Это позволяет измерять информационный вес любого состояния.

6. **Связь с алгеброй:** `branch` (D) и `compress` (H) — это комбинаторные операции, которые масштабируют ёмкость на ±1 уровень. Они реализуются через `*D_ID` и `/D_ID` для скаляров.
