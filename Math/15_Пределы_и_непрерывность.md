# 15. Пределы и Непрерывность

## I. Теория

### И.1. Предел как внутренний механизм навигации

Дуальные числа (`14_Дуальные_числа.md`) ввели векторную компоненту (velocity). Для перехода от дискретного (Хребет) к континууму (Поле) вводится концепция **Предела** — внутреннего механизма навигации системы.

**Сходимость:** последовательность `{aₙ}` сходится к **L**, если расстояние между `aₙ` и **L** становится меньше любой наперёд заданной величины.

### И.2. Два аттрактора

В системе два главных предельных состояния:

1. **Полнота (Π)** — Предел Ветвления:

   ```
   lim Dⁿ(Id) = Π,    n → ∞
   ```

   Абсолют, точка сходимости всех ветвящихся линий.

2. **Потенциал (Ω)** — Предел Сжатия:

   ```
   lim (a : Dⁿ(Id)) = Ω,    n → ∞
   ```

   Исчезновение Формы, сохранение Возможности.

### И.3. Непрерывность операций

Операции системы (`: Ω`, `: D(Id)`, `⊕`, `⊗`) **непрерывны** — не создают разрывов в ткани Носителя:

```
lim (xₙ : Ω) = (lim xₙ) : Ω
```

Если последовательность приближается к **a**, то ветвлённые значения приближаются к `D(a)`.

### И.4. Бесконечно малое ε

Бесконечно малое из дуальных чисел формализуется через предел:

```
ε = lim Δt,    Δt → Ω
```

Условие `ε² = Ω` означает, что квадрат бесконечно мал пренебрежимо мал по сравнению с ним самим:

```
lim (Δt² : Δt) = Ω
```

Система отбрасывает «шум второго порядка», оставляя чистую динамику.

### И.5. Полный набор пределов операций

| Операция | Предел | Результат |
|---|---|---|
| **Ветвление** | `lim Dⁿ(Id)` | `Π` |
| **Сжатие** | `lim (a : Dⁿ(Id))` | `Ω` |
| **Непрерывность D** | `lim D(xₙ)` | `D(lim xₙ)` |
| **Непрерывность H** | `lim H(xₙ)` | `H(lim xₙ)` |
| **ε** | `lim Δt` при `Δt → Ω` | Бесконечно малое |

### II.2. Состояния с поддержкой пределов

#### II.2.1. `SpineLevel` (`src/core/states/spine.py`)

| Метод | Математика | Реализация |
|-------|-----------|-----------|
| `branch()` | `Hⁿ⁺¹ = D(Hⁿ)` | `depth + 1.0` |
| `compress()` | `Dⁿ(Id) : D(Id) = Dⁿ⁻¹(Id)` | `max(0, depth − 1.0)` |
| `scale(exp)` | `(Dⁿ(Id))^b = D^(n·b)(Id)` | `depth * exp` |

**Функции:** `spine_level(depth)`, `root(value, degree)`, `is_spine_level(value)`.

**Граница:** `depth < 0` → `ValueError`.

## III. Зачем

### Инженерная аргументация

1. **Пределы — контракты для трансцендентных функций.** `exp()`, `ln()`, `sqrt()` используют `PI` и `OMEGA` как границы переполнения/неопределённости. Без контрактов пределов эти функции не имеют точек перегиба.

2. **Непрерывность — гарантия стабильности итераций.** `continuity_D` и `continuity_H` вычисляют погрешность `|f(lim) − f(x_tail)|`. При `≈ 0` — операция стабильна. При `> 0` — итерация расходится. Это используется в циклах вычислений для обнаружения расхождения.

3. **`SpineLevel` — логарифмическая шкала глубины.** `depth` вместо `2^depth` позволяет работать с бесконечной рекурсией без переполнения float. `value()` возвращает `2^depth` только при необходимости.

4. **Диспетчеризация в `branch()`/`compress()` — единая точка входа.** Все типы состояний (`SpineLevel`, `DualNumber`, `ComplexState`, `Fullness`, `FullnessShare`, `ParticipationRatio`) обрабатываются через `_complex_fullness`, `_complex_state`, `_dual_number`, `_fullness_share`, `_spine_level`, `_participation_ratio` — pattern-matching через `_type` конструкторы.

5. **`vectorization.map_scalar_or_vector` — прозрачная работа с тензорами.** Скалярные операторы (`branch`, `compress`, `divide`, `power`) автоматически поднимаются до векторов через `map_scalar_or_vector`.
