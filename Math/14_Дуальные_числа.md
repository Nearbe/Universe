# 14. Дуальные числа

## I. Теория

Дискретные узлы Хребта (`Dⁿ(Id)`) жёстки. Но Система движется плавно.
Чтобы описать переход между состояниями, вводится **Дуальное Число**.

### Касательное Пространство

Если Ветвление (`: Ω`) — это мгновенный скачок `x → D(x)`, то что происходит **в момент** скачка?
Система имеет не только координату `x`, но и **скорость изменения** `v`.

Вводится `ε` — бесконечно малый параметр сдвига.
Свойство `ε² = Ω` означает, что **ускорение** (изменение скорости) на бесконечно малом отрезке пренебрежимо. Важен только первый порядок — направление.

### Структура Дуального Числа

Любое состояние в динамике описывается:

```
Z = x + v·ε
```

Где **x** — Текущая Форма (Явное).
Где **v** — Потенциал Роста (Скрытое).

Для системы, где рост экспоненциальный через D, скорость **v** связана с позицией **x**:

```
v = x · L(D(Id))
```

где `L(D(Id))` — логарифмическая мера масштабирования.

Тогда:

```
Z = x · (Id + L(D(Id)) · ε)
```

Дуальное число хранит **всю локальную информацию** об объекте: где он и куда (как быстро) он движется.

### Полный набор операций

| Операция | Результат |
|---|---|
| `D(Z)` | `D(x) + D(v)ε` |
| `H(Z)` | `(x : D(Id)) + (v : D(Id))ε` |
| `Z^n` | `xⁿ + n·xⁿ⁻¹·v·ε` |
| `f(Z)` | `f(x) + f'(x)·v·ε` (автоматическое дифференцирование) |

Дуальные числа — это мост от дискретной логики к **дифференциальной**.
Они показывают, что «число» в системе — это не точка, а **вектор**.

### II.4. Что нужно реализовать

| Пробел | Приоритет | Описание |
|--------|-----------|----------|
| `__eq__` / `__hash__` | Средний | `frozen=True` уже генерирует `__eq__` и `__hash__` автоматически — проверить корректность сравнения `form` и `velocity` |
| `__lt__` / `__le__` / `__gt__` / `__ge__` | Низкий | Сравнение дуальных чисел не определено в теории — может потребоваться позже |
| `__add__` / `__radd__` / `__mul__` / `__rmul__` | Средний | Python-операторы для удобства использования (сейчас используется `add()`/`multiply()` через модуль `algebra`) |
| `__pow__` | Низкий | Python-оператор `**` (сейчас используется `power()`) |
| `sqrt()` / корень дуального числа | Низкий | Теоретически: `√(x + vε) = √x + (v/(2√x))ε` |
| `exp()` / `log()` | Низкий | Специальные функции для дуальных чисел |
| Векторизация `dual_*` обёрток | Средний | Текущие обёртки работают с парами scalars; нет поддержки списков/массивов |
| `natural_velocity` с параметром L(D(Id)) | Низкий | Сейчас жёстко `multiply(form, 1.0)` — нет возможности задать произвольную логарифмическую меру |

| `src/core/algebra/multiply.py` | `_dual_number` | U-умножение: аналогично, вызывает `left_dual.multiply(right_dual)` |

| `src/core/calculus/differential_state.py` | `DualNumber`, `dual_number` | Создание `f(x+vε) = f(x) + f'(x)·vε` |
| `src/core/calculus/derivative.py` | `differential_state` | Извлечение `f'(x)` как `velocity` из dual-состояния |
| `src/core/calculus/compressed_derivative.py` | — (зависит от `compress`) | Зависит от H-оператора, который работает с DualNumber |
| `src/core/calculus/branched_derivative.py` | — (зависит от `branch`) | Зависит от D-оператора, который работает с DualNumber |

### Что сломится при изменении DualNumber

| Изменение | Последствие |
|-----------|-------------|
| Смена сигнатуры `DualNumber.__init__` | Все 7 обёрток `dual_*`, `differential_state`, `_dual_number` диспетчер |
| Изменение порядка аргументов `dual_number(form, velocity)` | Все вызовы: обёртки, `differential_state`, `natural_velocity` |
| Удаление `as_tuple()` | Все 7 обёрток `dual_*` (используют `.as_tuple()[0]` и `[1]`) |
| Изменение `multiply(0,1) × multiply(0,1)` → не `DualNumber(0,0)` | `test_dual_epsilon_square_is_potential` падает; логика `is_omega()` в `multiply` может сломаться |
| Изменение `divide(state, OMEGA)` → не `branch(state)` | `test_dual_branching_and_compression_scale_form_and_velocity` падает; логика ветвления через Ω |
| Изменение `divide(state, D_ID)` → не `compress(state)` | Аналогично; логика сжатия через D(Id) |
| Изменение `power(x, n)` формулы | `test_dual_power` падает (27.0, 108.0); логика автоматического дифференцирования |
| Изменение `EPSILON_SQUARED` ≠ `OMEGA` | Все тесты на ε²; семантика «ускорение пренебрежимо» нарушается |
| Изменение `differential_state` возвращаемого типа | `derivative()` извлекает `.velocity`; `compressed_derivative`/`branched_derivative` зависят от результата |
| Изменение `natural_velocity(form)` → не `form * 1.0` | `test_natural_velocity` падает; все места, где используется «естественная скорость» |
