# 27. Вероятность

## I. Теория

Комбинаторика ([26](26_Комбинаторика.md)) посчитала ветви. Но пока система не измерила результат, она не знает, в какой ветви находится.
Вероятность — это **мера неопределённости**.

## Доля Ветвей

В системе **𝕌** вероятность — это не абстрактное число от **Ω** до **Id**.
Это **доля ветвей**, удовлетворяющих условию, среди всех возможных.

```
P(A) = (Число ветвей A) : (Все ветви)
```

Связь с долями ([12](12_Алгебра_процентов.md)):

```
p% = P(A) ⊗ Π
```

Где **Π** — Полнота всех исходов.

## Биномиальное Распределение

При **n** Актах (`: Ω`) с равным выбором ветвей:

```
P(k успехов) = C(n, k) : Dⁿ(Id)
```

Знаменатель `Dⁿ(Id)` — общее число ветвей.
Числитель — сколько из них ведут к результату **k**.
Вероятность — это просто **статистика порождения**.

## Нормальное Распределение

При `n → ∞` биномиальное распределение стремится к колокообразной кривой.
В логарифмической шкале ([08](08_Логарифм.md)) это означает:
Большинство путей концентрируются вокруг «золотой середины» (баланса ветвей).
Крайние состояния экспоненциально редки.

## Условная Вероятность

`P(A|B)` — вероятность **A**, если мы уже знаем, что система прошла через **B**.
В фрактале это означает сужение просмотра до конкретной подветки.
Мы отбрасываем всё, что несовместимо с историей **B**.

## Дискретность Системы

В **𝕌** нет «непрерывного незнания».
Состояние системы — всегда **конкретная точка** на конкретном уровне.
Вероятность описывает не «размытость» реальности, а **ограниченность наблюдателя**.
Для самой системы (Вселенной) вероятность всегда **Id** — она всегда «знает», где находится.

## Закон Больших Чисел

При большом числе испытаний среднее значение стабилизируется.
В системе **𝕌** это означает:
Хотя отдельный путь случаен, **распределение по всем путям** жёстко задано структурой дерева.
Хаос на микро-уровне порождает Порядок на макро-уровне.

## Полный набор операций

| Операция | Формула |
|---|---|
| **Вероятность** | `P(A) = (ветви A) : (все ветви)` |
| **Биномиальная** | `P(k) = C(n,k) : Dⁿ(Id)` |
| **Условная** | `P(A\|B) = P(A∩B) : P(B)` |
| **Полная** | `Σ P(A_i) = Id` |
| **Доля** | `p% = P(A) ⊗ Π` |

## Физический Смысл

Вероятность — это не «шум».
Это **структура порождения**.
Каждый Акт (`: Ω`) раздваивает реальность. Вероятность показывает, какая доля реальности приходится на интересующий нас исход.

## Статус

Вероятность в **𝕌** — это доля голоса в хоре ветвей.
Она объективна: зависит только от геометрии дерева решений, а не от субъективных ожиданий.

### II.1. Ключевые типы и таблицы

#### BranchProbability

```
src/core/probability/branch_probability.py
```

| Поле / метод | Тип | Описание |
|---|---|---|
| `matching_branches` | `int` | Число ветвей, удовлетворяющих условию A |
| `total_branches` | `int` | Общее число ветвей `Dⁿ(Id)` |
| `value` | `object` | `matching_branches : total_branches` через `divide()` |
| `to_fullness_share()` | `FullnessShare` | `P(A) ⊗ Π` — доля в полноте |

**Пост-условия (validation):**
- `matching_branches >= 0` — бросает `ValueError` иначе
- `total_branches > 0` — бросает `ValueError` иначе
- `matching_branches <= total_branches` — бросает `ValueError` иначе

### II.2. Функции / классы

#### II.2.1. `branch_probability(matching, total) → BranchProbability`

**Зависимости:** None (конструктор с валидацией)

**Инвариант:** Создаёт `BranchProbability` с проверкой пост-условий.

#### II.2.2. `binomial_probability(successes, acts) → BranchProbability`

**Формула:** `P(k) = C(n, k) : Dⁿ(Id)`

**Зависимости:**
- `combinatorics.binomial_coefficient(n, k)` — числитель `C(n,k)`
- `combinatorics.paths_count(n)` — знаменатель `Dⁿ(Id)`

**Пограничные случаи:**
- `successes < 0` или `acts < 0` или `successes > acts` → `branch_probability(0, 1)` (невозможное событие)

#### II.2.3. `conditional_probability(intersection, condition) → object`

**Формула:** `P(A∩B) : P(B)`

**Зависимости:**
- `algebra.divide` — U-деление

**Пограничные случаи:**
- `condition.matching_branches == 0` → бросает `ValueError`

#### II.2.4. `total_probability(parts) → object`

**Формула:** `Σ matching : total` для равного `total`

**Зависимости:**
- `algebra.divide` — U-деление
- `sum()` — встроенная сумма числителей

**Инварианты:**
- Пустой `parts` → `IMPOSSIBILITY` (0.0)
- Все `parts` должны иметь одинаковый `total_branches` — бросает `ValueError` иначе

#### II.2.5. `universe_probability() → float`

**Результат:** `CERTAINTY` (1.0) — система всегда «знает» своё состояние.

#### II.2.6. `binomial_probability(k, n) → object` (compat wrapper)

**Зависимости:** `branch_probability.binomial_probability(k, n).value`

**Назначение:** Возвращает числовое значение (float) вместо `BranchProbability`.

## III. Зачем

Вероятность в **𝕌** — это инженерный механизм подсчёта доли ветвей в бинарном дереве решений.

**Инженерная аргументация:**

1. **Бинарное дерево как фундамент:** Каждый акт ветвления `: Ω` удваивает пространство состояний. `Dⁿ(Id) = 2ⁿ` — это точное количество листьев на глубине `n`. Вероятность — это отношение числа листьев, удовлетворяющих условию, к общему числу листьев.

2. **Связь с комбинаторикой:** Биномиальное распределение `C(n,k) : Dⁿ(Id)` — это не абстрактная формула, а счёт путей в бинарном дереве. `C(n,k)` путей ведут к `k` правым ветвям из `n` актов.

3. **Связь с FullnessShare:** Результат вероятности `P(A)` может быть преобразован в `FullnessShare(P(A))` для выражения доли в Π. Это позволяет вероятностным результатам участвовать в U-алгебре (`⊕`, `⊗`, `:`).

4. **Sweep-анализ:** `compute_sweep()` и `compute_jump_events()` используют вероятностную интерпретацию occupancy-ставок (процент элементов поля, превышающих порог) для обнаружения скачков — точек, где распределение резко меняет форму.

5. **Логарифмическая ось:** `ln(x)` и `log2(x)` обрабатывают границы Ω (−∞) и Π (+∞) как алгебраические константы, что позволяет работать с вероятностями в логарифмической шкале без потери смысла.

6. **Дискретность:** Вероятность в **𝕌** всегда имеет точное рациональное представление (`matching : total`). Нет «непрерывного незнания» — только ограниченность наблюдателя в учёте всех ветвей.
