# 21. Интегральное Исчисление

## I. Теория

Дифференцирование ([20](20_Дифференциальное_исчисление.md)) извлекало скорость изменения.
Интеграл — парная операция. Это **накопление** величины по шкале удвоений.

### Неопределённый Интеграл

```
∫ f(x) dx = F(x) ⊕ C
```

Где `F'(x) = f(x)`.
Интегрирование — это **обратный путь к исходному состоянию**. Как `a^Ω = Id` возвращает систему к Единству ([07](07_Корни.md)), так интеграл восстанавливает целое по его скорости.

### Реакция на Ветвление

Как ведёт себя интеграл при масштабировании (`x → D(x)`)?

```
∫ f(D(x))dx = (∫ f(u)du) : D(Id)
```

Интеграл от ветвлённой функции сжимается.
Это парное правило к дифференцированию: если функция растёт через **D**, то площадь под ней «сжимается» — тот же результат достигается на меньшем отрезке.

### Связь с Логарифмической Шкалой

Переход к координатам глубины (`ℓ = L(x)`):

```
∫ f(x)dx = ∫ f(D^ℓ(Id)) · L(D(Id)) · D^ℓ(Id) dℓ
```

Интеграл в логарифмических координатах — это взвешенный по `D^ℓ(Id)` интеграл по шкале удвоений ([08](08_Логарифм.md)).
Мы суммируем не по «длине», а по **тактам Ветвления**.

### Определённый Интеграл

```
∫ₐᵇ f(x)dx = F(b) ⊖ F(a)
```

Разность первообразных — это **конечный результат накопления** между двумя состояниями системы.
Это мера того, сколько «эффекта» произвела функция при переходе от **a** к **b**.

### Полнота и Интеграл

```
∫ₐ^Π f(x)dx
```

Предел на горизонте ([10](10_Бесконечность.md)).
Интеграл сходится, если функция убывает быстрее, чем `Id : x`.
Для `D(x)` он расходится — что естественно: экспонента растёт быстрее любого полинома, и Полнота (**Π**) не может быть исчерпана суммированием.

### Полный набор операций

| Операция | Формула |
|---|---|
| **Неопределённый** | `∫ f(x)dx = F(x) ⊕ C` |
| **При ветвлении** | `∫ f(D(x))dx = (∫ f(u)du) : D(Id)` |
| **При сжатии** | `∫ f(x:D(Id))dx = (∫ f(u)du) ⊗ D(Id)` |
| **Определённый** | `∫ₐᵇ f(x)dx = F(b) ⊖ F(a)` |
| **Логарифм. коорд.** | `∫ f(D^ℓ) · L(D(Id)) · D^ℓ dℓ` |
| **К Полноте** | `∫ₐ^Π f(x)dx` — сходится если f убывает достаточно быстро |

### Статус

Интеграл — это суммарный эффект непрерывного применения масштабных преобразований.
Дифференцирование и Интегрирование — парные операции, как `: Ω` (Ветвление) и `: D(Id)` (Сжатие), как рост и возврат.

### II.2. Функции/классы интегрального исчисления

Все функции экспортируются из `src/core/calculus/__init__.py`.

#### II.2.1. `antiderivative(primitive_fn, point, *, constant=0.0) → object`

**Формула:** `F(x) ⊕ C`

**Реализация:** `add(primitive_fn(point), constant)`

**Зависимости:** `src/core/algebra/add.py::add`

**Семантика:** Возвращает значение первообразной в точке `point` плюс константу. Используется как подоперация для `definite_integral`.

**Параметры:**
- `primitive_fn: Callable[[object], object]` — функция первообразной (например, `cubic_antiderivative`)
- `point: object` — точка вычисления (число, SpineLevel, DualNumber и т.д.)
- `constant: object = 0.0` — аддитивная константа (по умолчанию 0.0)

#### II.2.2. `definite_integral(primitive_fn, lower, upper) → object`

**Формула:** `F(b) ⊖ F(a) = F(b) + (−F(a))`

**Реализация:** `add(primitive_fn(upper), -float(primitive_fn(lower)))`

**Зависимости:** `src/core/algebra/add.py::add`, встроенный `float()` и унарный `-`

**Семантика:** Вычисляет разность первообразной на верхней и нижней границах. Для вычитания используется `+ (−F(a))` через Python-арифметику, а не отдельная операция `⊖`.

**Параметры:**
- `primitive_fn: Callable[[object], object]` — функция первообразной
- `lower: object` — нижняя граница
- `upper: object` — верхняя граница

**Ограничения:** `lower` приводится к `float` перед применением унарного минуса. Если `primitive_fn` возвращает не-скаляр (DualNumber, ComplexState), `float()` вызовет `TypeError`.

#### II.2.3. `branched_integral(primitive_fn, point) → object`

**Формула:** `∫ f(D(x))dx = (∫ f(u)du) : D(Id)`

**Реализация:** `compress(primitive_fn(branch(point)))`

**Зависимости:**
- `src/core/algebra/branch.py::branch` — `D(value) = value * D_ID` (масштабирование на 2.0 для скаляров)
- `src/core/algebra/compress.py::compress` — `H(value) = value / D_ID` (сжатие на 2.0 для скаляров)

**Семантика:** Применяет ветвление к точке (`branch`), вычисляет первообразную, затем сжимает результат (`compress`). Это эквивалент `: D(Id)` на скалярах.

**Параметры:**
- `primitive_fn: Callable[[object], object]` — функция первообразной
- `point: object` — точка вычисления

#### II.2.4. `compressed_integral(primitive_fn, point) → object`

**Формула:** `∫ f(x:D(Id))dx = (∫ f(u)du) ⊗ D(Id)`

**Реализация:** `branch(primitive_fn(compress(point)))`

**Зависимости:**
- `src/core/algebra/compress.py::compress` — `H(value) = value / D_ID`
- `src/core/algebra/branch.py::branch` — `D(value) = value * D_ID`

**Семантика:** Сжимает точку (`compress`), вычисляет первообразную, затем ветвит результат (`branch`). Это эквивалент `⊗ D(Id)` на скалярах.

**Параметры:**
- `primitive_fn: Callable[[object], object]` — функция первообразной
- `point: object` — точка вычисления

#### II.2.5. `integral_on_log_depth(fn, depth) → object`

**Формула:** `f(D^ℓ) · L(D(Id)) · D^ℓ`

**Реализация:** `multiply(multiply(fn(state), L(D_ID)), state)` где `state = depth.value()`

**Зависимости:**
- `src/core/states/spine.py::SpineLevel` — глубина рекурсии `Dⁿ(Id)`
- `src/core/operators/L.py::L` — логарифмическая глубина `L(x) = log₂|x|`
- `src/core/constants/constants.py::D_ID` — `D_ID = 2.0`
- `src/core/algebra/multiply.py::multiply` — U-умножение

**Семантика:** `depth` — объект `SpineLevel`. `depth.value()` возвращает `D_ID ** depth.depth` = `2^depth`. `L(D_ID)` = `log₂(2.0)` = `1.0`. Итог: `fn(2^depth) * 1.0 * 2^depth` = `fn(2^depth) * 2^depth`.

**Параметры:**
- `fn: Callable[[object], object]` — функция для интегрирования
- `depth: SpineLevel` — глубина на хребте

#### II.2.6. `power_antiderivative(exponent, point) → object`

**Формула:** `x^(b⊕Id) : (b⊕Id)` — первообразная `x^b`

**Реализация:** `divide(power(point, next_exponent), next_exponent)` где `next_exponent = add(exponent, IDENTITY_VELOCITY)` = `exponent + 1.0`

**Зависимости:**
- `src/core/algebra/power.py::power` — U-степень
- `src/core/algebra/divide.py::divide` — U-деление
- `src/core/algebra/add.py::add` — U-сложение
- `src/core/calculus/differential_state.py::IDENTITY_VELOCITY` — `1.0`

**Семантика:** Классическое правило степенной функции: `∫ x^b dx = x^(b+1) / (b+1)`. Использует `IDENTITY_VELOCITY = 1.0` как шаг по степени.

**Параметры:**
- `exponent: object` — показатель степени `b`
- `point: object` — точка вычисления

### Зависимости от `src/core/algebra/`

Все интегральные функции зависят от базовых алгебраических операций:

| Операция | Модуль | Используется в |
|---|---|---|
| `add` | `algebra/add.py` | `antiderivative`, `definite_integral`, `power_antiderivative` |
| `branch` | `algebra/branch.py` | `branched_integral` |
| `compress` | `algebra/compress.py` | `branched_integral`, `compressed_integral` |
| `divide` | `algebra/divide.py` | `power_antiderivative` |
| `multiply` | `algebra/multiply.py` | `integral_on_log_depth` |
| `power` | `algebra/power.py` | `power_antiderivative` |

**Критическая цепочка:** `branched_integral` → `branch` → `SpineLevel.branch()` / `DualNumber.branch()` / скаляр `* D_ID`. Аналогично `compressed_integral` → `compress` → `SpineLevel.compress()` / `DualNumber.compress()` / скаляр `/ D_ID`.

### Что сломится при изменении

1. **Изменение `D_ID` (2.0):** Сломает все масштабирование ветвления/сжатия в `branched_integral`, `compressed_integral`, `branched_derivative`, `compressed_derivative`, `branching_growth_velocity`, `SpineLevel`.
2. **Изменение `IDENTITY_VELOCITY` (1.0):** Сломает `power_antiderivative` (шаг по степени) и `power_derivative`.
3. **Изменение `definite_integral`:** Текущая реализация использует `-float(primitive_fn(lower))` вместо U-вычитания. Если `primitive_fn` возвращает не-скаляр — `TypeError`.
4. **Изменение `integral_on_log_depth`:** Зависит от `L(D_ID) = 1.0`. Если `D_ID` изменится, `L(D_ID)` изменится, и веса интеграла будут неверными.
