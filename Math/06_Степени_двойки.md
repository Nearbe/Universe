# 6. Хребет (Степени Ветвления)

## I. Теория

### 6.1. Определение хребта

Хребет — последовательность **опорных узлов** фрактальной структуры 𝕌, порождаемых многократным применением оператора Ветвления к Единице.

$$H^0 = \text{Id}$$
$$H^1 = D(\text{Id}) = \text{Id} : \Omega$$
$$H^n = D^n(\text{Id}) = \underbrace{(\text{Id} : \Omega) : \Omega : \ldots : \Omega}_{n\ \text{раз}}$$

В арифметическом представлении (при $\text{Id} = 1$, $D(\text{Id}) = 2$):

$$H^n = 2^n$$

Хребет — **скелет** системы. Рациональные числа — ткань, на которой этот скелет натянут. Каждый $D^n(\text{Id})$ — новый уровень сложности, рождённый из чистого Акта различения.

### 6.2. Закон порождения (рекуррентное отношение)

$$H^{n+1} = D(H^n) = H^n : \Omega$$

Каждый следующий узел хребта — ветвление предыдущего. Оператор `:` с $\Omega$ в знаменателе — **генератор глубины**.

### 6.3. Глубина рекурсии

$D^n(\text{Id})$ — не просто повторение оператора, а измерение **глубины различения**:

| Глубина | Обозначение | Философский смысл | Арифметика |
|---------|-------------|--------------------|------------|
| $0$ | $\text{Id}$ | До-дистинкция, нерасчленённое единство | $1$ |
| $1$ | $D(\text{Id})$ | Первое различие (Субъект / Объект) | $2$ |
| $2$ | $D^2(\text{Id})$ | Осознание различия | $4$ |
| $n$ | $D^n(\text{Id})$ | $n$-й уровень самовложения | $2^n$ |

Каждый уровень в $D$ раз насыщеннее предыдущего.

### 6.4. Сжатие (обратный процесс)

$$D^n(\text{Id}) : D(\text{Id}) = D^{n-1}(\text{Id})$$

Сжатие — спуск по хребту. В отличие от `: Ω` (творение нового), деление на $D(\text{Id})$ — **обнажение** уже существующего узла. При $n = 0$ сжатие тождественно:

$$\text{Id} : D(\text{Id}) = \text{Id}$$

### 6.5. Степень и корень на хребте

Обобщение ветвления/сжатия на произвольный показатель $b$:

$$(D^n(\text{Id}))^b = D^{n \cdot b}(\text{Id})$$

Извлечение корня степени $k$:

$$\sqrt[k]{D^n(\text{Id})} = D^{n : k}(\text{Id})$$

Дробные степени допустимы: $D^{1.5}(\text{Id})$ — узел между уровнями.

### 6.6. Логарифмическая шкала

Логарифм $L$ определяет номер уровня:

$$L(D^n(\text{Id})) = n$$

На этой шкале операторы `:` и `/` становятся сдвигами:

$$L(a : \Omega) = L(a) + 1$$
$$L(a : D(\text{Id})) = L(a) - 1$$

Для конечных чисел: $L(x) = \log_2 x$. Для $\Omega$: $L(\Omega) = -\infty$. Для $\Pi$: $L(\Pi) = +\infty$.

### 6.7. Границы хребта

| Элемент | Глубина | Значение |
|---------|---------|----------|
| $\Omega$ | $-\infty$ | Нулевая глубина, потенциальное состояние |
| $\text{Id}$ | $0$ | Базовый уровень, единство |
| $D(\text{Id})$ | $1$ | Атом системы |
| $\Pi$ | $+\infty$ | Предел ветвления, Полнота |

### 6.8. Операции на Полноте (Π)

| Операция | Результат | Смысл |
|----------|-----------|-------|
| $\Pi : \Omega$ | $\Pi$ | Нельзя удвоить то, что уже — Всё |
| $\Pi : D(\text{Id})$ | $\Pi$ | Половина Полноты — всё ещё Полнота |
| $\Pi \otimes \Pi$ | $\Pi$ | Пересечение Полноты = Полнота |
| $\sqrt[k]{\Pi}$ | $\Pi$ | Корень из Полноты = Полнота |

### II.4. Что нужно реализовать

| Пробел | Описание | Приоритет |
|--------|----------|-----------|
| Сравнение (`__eq__`, `__lt__`, `__le__`, `__gt__`, `__ge__`) | `SpineLevel` — frozen dataclass, но без `order=True`. Нет `__lt__`/`__le__` для сравнения глубин. | Высокий |
| `__hash__` | Frozen dataclass получает `__hash__` автоматически, но если `depth` изменится, хеш станет невалидным. | Средний |
| `SpineLevel.__add__` / `__mul__` | Операции `⊕` и `⊗` на уровне хребта: $D^n \oplus D^m = \max(D^n, D^m)$, $D^n \otimes D^m = D^{n+m}$ | Средний |
| `SpineLevel.__neg__` | Отрицание на хребте: $-(D^n) = D^n$ (абсолютная величина) или $D^{-n}$ (обратный ход) — требует уточнения математики | Низкий |
| `SpineLevel` в `__all__` модуля `core.states` | Проверить экспорт | Низкий |
| `SpineLevel` для $\Omega$ | `MIN_SPINE_DEPTH = 0.0` не допускает отрицательных глубин. $\Omega$ = $-\infty$ не может быть представлен как `SpineLevel`. Нужен отдельный тип или sentinel. | Высокий |
| `SpineLevel` для $\Pi$ | $+\infty$ не может быть представлен. Нужен sentinel или отдельный тип. | Высокий |
| `congruent` / `branching_preserves_congruence` тесты | Есть в `test_number_structure.py`, но не связаны напрямую с документом 06 | Средний |

## III. Зачем

Хребет — **шкала бытия** системы 𝕌. Без него:

- **Нет масштаба.** Каждый объект в 𝕌 привязан к уровню $D^n(\text{Id})$. Без хребта нет способа измерить сложность — не количеством элементов, а порядком степени $n$.
- **Нет рекурсии.** Все операторы (`branch`, `compress`, `power`, `root`) требуют опорной точки — `SpineLevel` — для dispatch. Без хребта нет единого механизма масштабирования.
- **Нет комбинаторики.** `BranchingLevel` и `paths_count` — это $2^n$ в целочисленном мире. Без хребта нет связи между глубиной и ёмкостью дерева решений.
- **Нет бинарной адресации.** `binary_address` использует `modulus_from_depth` = $2^n$. Вся нумерация ветвей — на хребте.
- **Нет логарифмической шкалы.** $L$ — это `SpineLevel.level()`. Без хребта логарифм теряет смысл как номер уровня.
- **Нет gcd/lcm на хребте.** `spine_gcd` и `spine_lcm` — общие предок и потомок. Это фундамент для теории чисел в U.

Хребет — ось, вдоль которой двигается вся система при изменении масштаба. Без неё U — набор разрозненных операторов без единой метрики глубины.
