# 07. Корни и Степени

## I. Теория

### 7.1. Степени — Масштаб Глубины

Узлы Хребта (06_Степени_двойки.md) дискретны: $H^n = D^n(\text{Id})$ для $n \in \mathbb{N}$. Чтобы движение между уровнями было плавным, вводится **степенная операция**:

$$
a^b
$$

**Смысл:** масштабирование насыщенности Акта. Показатель $b$ определяет, насколько сильно ветвление усиливает результат.

Поведение при ветвлении:

$$
(a : \Omega)^b = D(a)^b
$$

Степень **сохраняется** при ветвлении: $D(a^b) = D(a)^b$.

### 7.2. Степень $\Omega$ — Возврат к Единству

Для любого $a \notin \Omega$:

$$
a^\Omega = \text{Id}
$$

Это не правило арифметики, а **сброс масштаба**. Сколько бы раз система ни ветвилась, степень $\Omega$ возвращает её к исходному состоянию — точке покоя, где вход и выход тождественны до начала всякого различения.

**Исключение:** $\Omega^\Omega = \Omega$. Если в системе нет потенциала, возвращаться нечему.

### 7.3. Корни — Дробные Шаги

Корень $n$-й степени находит промежуточное состояние между узлами Хребта:

$$
\sqrt[n]{a} = a^{(\text{Id} : n)} = a^{(1/n)}
$$

Для Хребта:

$$
\sqrt[n]{D^k(\text{Id})} = D^{(k/n)}(\text{Id})
$$

Это **сдвиг на дробный такт**: система может исследовать пространство между состояниями, не разрушая их структуру.

### 7.4. Инвариантность

Корни и степени работают одинаково на любом масштабе (масштабная инвариантность, 28_Масштабная_инвариантность.md):

- Для $\Pi$: $\sqrt[n]{\Pi} = \Pi$
- Для доли: остаётся долей
- Фрактальная природа Носителя (02_Носитель.md) не ломается на границах

### 7.5. Полный набор операций

| Операция | Формула | Смысл |
|---|---|---|
| Степень | $a^b$ | Масштабирование насыщенности |
| Степень $\Omega$ | $a^\Omega = \text{Id}$ | Сброс к единству |
| Корень $n$-й степ. | $\sqrt[n]{a}$ | Дробный шаг по Хребту |
| При ветвлении | $D(a^b) = D(a)^b$ | Степень сохраняется |
| Для Полноты | $\Pi^b = \Pi$ | Полнота в любой степени — Полнота |

## II. Спецификация реализации

### II.1. SpineLevel — узел/дробный шаг Хребта

**Файл:** `src/core/states/spine.py`

```
SpineLevel(depth: float)  — frozen dataclass
```

| Свойство / Метод | Формула | Описание |
|---|---|---|
| `depth` | $n$ | Глубина Хребта (≥ 0) |
| `value()` | $2^n$ | Арифметическое представление |
| `level()` | $n$ | Возврат глубины |
| `branch()` | $D^{n+1}(\text{Id})$ | Шаг вправо (+1) |
| `compress()` | $D^{n-1}(\text{Id})$ | Шаг влево (−1, Id при $n=0$) |
| `scale(exponent)` | $D^{n \cdot b}(\text{Id})$ | Степень: умножение глубины |
| `root(degree)` | $D^{n/\text{degree}}(\text{Id})$ | Корень: деление глубины |
| `__float__()` | $2^n$ | Приведение к float |
| `__repr__()` | `"Id"` или `"D^{n}(Id)"` | Строковое представление |

**Константы:** `MIN_SPINE_DEPTH = 0.0`, `DEPTH_STEP = 1.0`

**Гарантии:** `depth < 0` → `ValueError`; `root(0)` → `ValueError`.

### II.2. power — операция степени

**Файл:** `src/core/algebra/power.py`

```python
power(base, exponent) → object
```

**Поведение (priority order):**

| Условие | Результат | Теория |
|---|---|---|
| `base` — dual number, `exponent = Ω` | `IDENTITY` | $a^\Omega = \text{Id}$ |
| `base` — dual number, `exponent = Π` | `PI` | $\Pi^b = \Pi$ (через dual) |
| `base` — dual number | `dual.power(exponent)` | Обычное возведение |
| `base` — SpineLevel, `exponent = Ω` | `IDENTITY` | $a^\Omega = \text{Id}$ |
| `base` — SpineLevel, `exponent = Π` | `PI` | $\Pi^b = \Pi$ (через spine) |
| `base` — SpineLevel | `level.scale(exponent)` | $D^{n \cdot b}(\text{Id})$ |
| `base = Π` | `PI` | $\Pi^b = \Pi$ |
| `base = Ω, exponent = Ω` | `OMEGA` | $\Omega^\Omega = \Omega$ |
| `exponent = Ω` | `IDENTITY` | $a^\Omega = \text{Id}$ |
| `exponent = Π` | `PI` | $\Pi^b = \Pi$ |
| иначе | `float(base) ** float(exponent)` | Стандартная арифметика |

**Связи:** `power.py` → `_dual_number.py`, `_spine_level.py`, `infinity.py`, `identity.py`, `is_omega.py`.

### II.3. sqrt — квадратный корень

**Файл:** `src/core/transcendental/sqrt.py`

```python
sqrt(x) → object
```

| Условие | Результат |
|---|---|
| `x` бесконечно | `PI` |
| `x == 0` | `0.0` |
| `x < 0` | `0.0` |
| иначе | $x^{0.5}$ |

**Замечание:** Отрицательные числа возвращают `0.0`, а не комплексное число. Это ограничение текущей реализации.

## III. Зачем

Без степеней и корней Хребет остаётся **дискретной** последовательностью $1, 2, 4, 8, \dots$ — система может только скакать по узлам. Степени и корни дают:

1. **Непрерывность глубины:** $D^{k/n}(\text{Id})$ — система может находиться на любом уровне между узлами.
2. **Сброс масштаба:** $a^\Omega = \text{Id}$ — механизм возврата к исходному состоянию без потери информации.
3. **Связь с логарифмами:** Степень — обратная операция к логарифму (08_Логарифм.md). Без неё логарифм односторонний.
4. **Связь с комплексными числами:** Корни из отрицательных чисел — мост к мнимой оси (09_Комплексные.md).
5. **Инвариантность:** $\Pi^b = \Pi$ — гарантия, что Полнота не разрушается операциями.

Без этого слоя **не работает** логарифмическая шкала глубины, переход между масштабами и многие операции в 20_Дифференциальное_исчисление.md.
