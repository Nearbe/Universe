# 23. Соленоид (𝒮) — Спецификация

## I. Теория

### Обратный предел
Соленоид — пространство всех согласованных историй ветвления:
```
𝒮 = lim← (𝕌 : Ω ← 𝕌 : Ω ← 𝕌 : Ω ← ...)
```
Точка соленоида — вся траектория масштабирований, уходящая в прошлое:
```
z₀ = D(z₁) = D²(z₂) = ... = Dⁿ(zₙ)
```

### Альтернативное определение
```
𝒮 = {(z₀, z₁, z₂, ...) ∈ 𝕌^ℕ | zₙ = D(zₙ₊₁)}
```

### Граф как Соленоид
- **Узел:** пузырь — локальное состояние с массой и фазой
- **Ребро:** взаимодействие — сила связи между пузырями
- **Матрица смежности M:** `M_ij = Ω` (нет связи), `M_ij ≠ Ω` (течёт энергия)
- **Степень вершины:** `k_i` — число связей узла `i`

### Двоичная природа
На каждом шаге назад — выбор ветви `εₙ ∈ {Ω, D(Id)}`. История кодируется двоичной дробью `ξ = 0.ε₀ε₁ε₂…`. Каждая точка соленоида — фаза + бесконечная двоичная последовательность.

### Сдвиг Бернулли (Действие `: Ω`)
Акт ветвления сдвигает двоичную запись влево. Первый бит отбрасывается. Детерминированный хаос с показателем Ляпунова `λ = L(D(Id)) > Ω`.

### Спектральная теория
- **Собственные значения:** `M ⊗ v = λ ⊗ v`
- **Лапласиан:** `L = D_deg ⊖ A`
- `λ₂ > Ω` ⟹ сеть связна; `λ₂ ≈ Ω` ⟹ почти разваливается

### Типы сетей
| Тип | Формула | Свойство |
|---|---|---|
| Регулярные | `k_i = const` | Предсказуемые |
| Случайные | `P(k) ~ e^(⊖k)` | Хаотичные |
| Безмасштабные | `P(k) ∝ k^(⊖γ)` | Хабы |
| Иерархические | Уровень 0 → 1 → 2 | Фрактальная самоподобность |

### Метрики связности
| Метрика | Смысл |
|---|---|
| Длина пути | Среднее расстояние |
| Кластеризация | Локальная связность |
| Степенная | Влиятельность узла |
| Посредническая | Узел-мост |

### Динамика сети
```
Δ_i(n+1) = Δ_i(n) − Γ·Δ_i(n) + Σw_ij·Δ_j(n)
```
- **Перколяция:** порог связности `p_c ≈ 0.5927` (квадратная решётка)
- **Правило Хебба:** `Δw_ij ∝ pre_i ⊗ post_j`

### Метрика близости
Две точки близки, если совпадают фазы И двоичные истории на многих первых шагах. Происхождение важнее текущего вида.

### Связь с p-адическими числами
`ℤ_D = lim← (𝕌 : Dⁿ(Id))` — соленоид = геометрическая реализация D(Id)-адических чисел.

### Связь с Полнотой
Единичная окружность (`|z| = Id`) — срез Полноты. Доли живут на этом срезе. Π — горизонт, к которому сходятся все лучи.

#### `SweepResults` (`src/core/sweep/sweep_results.py`)

```python
class SweepResults:
    __slots__ = ("thresholds", "occupancy_rates", "jump_events", "jump_count")
```
| Поле | Тип | Описание |
|---|---|---|

| `occupancy_rates` | `CoreMatrix` | Матрица `[thresholds × classes]` — % активных состояний |
| `jump_events` | `list[tuple[float, int, float, float, float]]` | `(threshold, class_id, before, after, change)` |
| `jump_count` | `int` | Количество скачков |

#### `compute_sweep` (`src/core/sweep/compute_sweep.py`)
**Вход:** объект с атрибутами `delta_min`, `delta_max`, `config` (dict), `symbol_delta_fields` (iterable).
**Выход:** `SweepResults`.

**Алгоритм:**
1. Извлечь параметры: `sweep_min`, `sweep_max`, `sweep_step`, `jump_threshold`
2. Сгенерировать пороги: `compute_thresholds(sweep_min, sweep_max, sweep_step)`
3. Для каждого порога и каждого поля: `active_count / total × 100`
4. Транспонировать матрицу: `per_class_rates = occupancy_rates.T`
5. Найти скачки: `compute_jump_events(thresholds, per_class_rates, jump_threshold)`
6. Вернуть `SweepResults`

**Дефолты:** `sweep_min=-5.6`, `sweep_max=5.6`, `sweep_step=0.1`, `jump_threshold=1.0`

#### `compute_thresholds` (`src/core/sweep/compute_thresholds.py`)
```
count = int((sweep_max - sweep_min) / sweep_step) + 1
thresholds = [sweep_min + i × sweep_step for i in range(count)]
```

#### `compute_jump_events` (`src/core/sweep/compute_jump_events.py`)
**Вход:** `thresholds`, `occupancy_rates[class_id][threshold_idx]`, `jump_threshold`.
**Выход:** `(events, count)`.

**Алгоритм:** Для каждой пары `(class_id, threshold_idx)`:
```
change = |rates[threshold_idx + 1] - rates[threshold_idx]|
if change > jump_threshold → event = (threshold, class_id, before, after, change)
```

#### `rg_aware_sweep` (`src/core/sweep/rg_aware_sweep.py`)
Генерация порогов с renormalization group коррекцией:
```
t = sweep_min + (sweep_max - sweep_min) × level/n_levels + k × 2^(-level)
```
где `level ∈ [0, n_levels)`, `k ∈ [-4, 4]`.

#### `inverse_jump_analysis` (`src/core/sweep/inverse_jump_analysis.py`)
Traceback скачков:
```
trace = [event[0] / 2^k for k in range(n_back)]
```

#### `sweep_process_class` (`src/core/sweep/sweep_process_class.py`)

## III. Зачем

### Инженерная аргументация

1. **Соленоид как метрическое пространство** — `SolenoidPoint` + `solenoid_distance` обеспечивают метрику на пространстве историй. Это база для кластеризации, поиска ближайших соседей и сравнения паттернов с учётом происхождения (shared binary prefix).

2. **Sweep как визуализация Δ-поля** — `compute_sweep` вычисляет occupancy rates по пороговой сетке для каждого класса. `compute_jump_events` находит пороги, где occupancy резко меняется — это точки бифуркации в пространстве состояний.

3. **SpineLevel как опорная шкала** — `SpineLevel` даёт дискретную шкалу `D^n(Id) = 2^n` для референции глубины. Все остальные состояния (Dual, Complex, FullnessShare) масштабируются относительно spine.

4. **DualNumber как дифференциальная арифметика** — `DualNumber` несёт форму и скорость. `PatternSynthesizer` использует dual-арифметику для интерполяции/блендинга/эволюции паттернов с сохранением производной.

5. **RG-aware sweep** — `rg_aware_sweep` добавляет коррекции `k × 2^(-level)` к порогам, обеспечивая мультишкальное покрытие без пропущенных масштабов.

6. **Inverse jump analysis** — `inverse_jump_analysis` вычисляет `event_threshold / 2^k` для追溯 скачков к их истокам на разных масштабах.

7. **Fractal pattern signature** — `fractal_pattern_signature` генерирует компактный профиль (64-битный) для сравнения паттернов. `fractal_similarity_score` объединяет Jaccard + Betti для многомерного сравнения.

8. **Uroboros cycle** — `UroborosState` обеспечивает дискретный цикл 0→1→...→9→0 с отображением в `[-1, 1]` и семантические категории (Ω, Id, Π, transition).
