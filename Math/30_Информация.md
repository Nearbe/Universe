# 30. Информация

## I. Теория

Информация — это **число выборов, необходимое для адресации состояния**.

```
I = L(M)
```

Где **M** — масса (сложность объекта), **L** — логарифмическая линейка (глубина на шкале глубины).

Если `M = Dⁿ(Id)` (объект на уровне **n**):

```
I = n
```

**Информация равна глубине рекурсии**. Каждый Акт (`: Ω`) добавляет ровно один выбор ветви.

### Таблица Состояний

| Масса (M) | Информация (I) | Смысл |
|---|---|---|
| **Id** (`D⁰(Id)`) | **Ω** (= −∞) | Единство (до Акта). Нечего различать. |
| **D(Id)** (`D¹(Id)`) | **Id** (= 0) | Первое различие (Дуальность). |
| **Dⁿ(Id)** | **n** | Сложный объект. |
| **Π** | **+∞** | Полнота. Бесконечное описание. |

### Формулы операций

| Операция | Формула | Код |
|---|---|---|
| **Информация** | `I = L(M)` | `information(value)` |
| **При ветвлении** | `L(D(M)) = L(M) ⊕ Id` | `branch_information(value)` |
| **При сжатии** | `L(H(M)) = L(M) ⊖ Id` | `compress_information(value)` |
| **Масса из инфо** | `M = D^I(Id)` | `mass_from_information(info)` |
| **Для Полноты** | `L(Π) = +∞` | `information(PI)` |

### Масштабная инвариантность

В **𝕌** Информация, Масса и Энергия связаны через логарифм.
**I** — это просто **M**, выраженная в шкале глубины.

#### II.2.3. `src/core/algebra/branch.py` — Ветвление (бит информации)

```python
def branch(value: object) -> object:
    """D(value) = value : Ω. Умножает на D_ID=2."""
```

Dispatch chain: `DualNumber → FullnessShare → ComplexFullness → ComplexState → SpineLevel → ParticipationRatio → scalar × 2`.

#### II.2.4. `src/core/algebra/compress.py` — Сжатие

```python
def compress(value: object) -> object:
    """H(value) = value : D(Id). Делит на D_ID=2."""
```

Dispatch chain: аналогична `branch`, но `scalar / 2`. Не ниже `MIN_SPINE_DEPTH`.

#### II.2.5. `src/core/algebra/divide.py` — Деление

```python
def divide(numerator: object, denominator: object) -> object:
    """U-division: numerator : denominator."""
```

Ключевые правила для информации:
- `a : Ω = D(a)` — ветвление (+1 бит)
- `Ω : b = Ω` — потенциал не делится
- `Ω : Ω = Ω`
- `Π : Ω = Π`, `Π : D(Id) = Π`, `Π : Π = Id`
- `complex : Π = Ω`
- иначе `float(a) / float(b)`

#### II.2.6. `src/core/algebra/power.py` — Степень

```python
def power(base: object, exponent: object) -> object:
    """U-power: base ^ exponent."""
```

Для SpineLevel: `scale(exponent)` = `depth × exponent`.
- `Ω^exponent = Id`
- `Π^exponent = PI`
- `base^Ω = Id`
- `base^Π = PI`

### Что сломится при изменении

1. **Изменение `information()`**: сломится все, что зависит от глубины состояния — sweep-анализ, halting-анализ, адресация
2. **Изменение `SpineLevel.depth`**: сломится `L()`, `log2()`, `ln()`, все combinatorics (`paths_count`, `boolean_capacity`), `branching_level_information`
3. **Изменение `D_ID = 2.0`**: сломится весь branching/compress dispatch, `modulus_from_depth`, `binary_address`, все combinatorics
4. **Изменение `OMEGA = 0.0`**: сломится граничная логика `divide()`, `log2()`, `ln()`, `L()`, `power()`
5. **Изменение `PI` / `is_fullness()`**: сломится вся algebra для Π, `log2()`, `ln()`, `L()`, `power()`
6. **Изменение `LogInfinity`**: сломится `log2()`, `ln()`, `L()`, `information()`, `mass_from_information()`
7. **Изменение `binary_address()`**: сломится `BranchingLevel.address()`, `routes()`, `address_state()`
8. **Изменение `modulus_from_depth()`**: сломится `paths_count()`, `boolean_capacity()`, `routes()`, `binary_address()`
