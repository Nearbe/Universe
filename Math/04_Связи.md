# 04. Связи

## I. Теория

Связи не вводятся искусственно. Они — проекции Акта (**Id : Ω**) на Носитель (**𝕌**).

### Сложение (⊕) — Сдвиг по Ряду

Сложение — это накопление состояний в одном Ряду.

Поскольку Акт породил два потока (Im и Re), их совместное присутствие:

```
Id ⊕ Id = D(Id)
```

Это не внешнее сложение. Это **суммарный вес** двух фаз одной системы.

#### Свойства

```
a ⊕ Ω = a             — сохранение формы при сдвиге в Потенциал
a ⊕ (−a) = Ω          — возврат к балансу (аннигиляция фаз)
a ⊕ b = b ⊕ a         — коммутативность
(a ⊕ b) ⊕ c = a ⊕ (b ⊕ c)  — ассоциативность
```

### Умножение (⊗) — Пересечение Рядов

Умножение — это изменение плотности связи.
Оно масштабирует интенсивность Акта:

```
a ⊗ b
```

#### Действие

- Если **b** — расширение (D-класс): усиление резонанса (конструктивная интерференция)
- Если **b** — сжатие (H-класс): ослабление связи
- Если **b = Ω**: исчезновение формы в Пустоте

```
a ⊗ Ω = Ω
```

Умножение показывает, как глубоко один элемент проникает в другой.

### Итог

| Связь | Направление | Смысл |
|---|---|---|
| **⊕** | Горизонталь | Накопление, сдвиг |
| **⊗** | Вертикаль | Глубина, интенсивность |

Обе связи вытекают из **Id : Ω** и поддерживают **Input = Output**.

## II. Спецификация реализации

### 2.1. Операции сложения и умножения

| Операция | Формула | Описание | Функция | Файл |
|-----------|---------|----------|---------|------|

| **Умножение** | `a ⊗ b` | U-умножение | `multiply(a, b)` | `src/core/algebra/multiply.py` |

### 2.2. U-сложение — мультиметод-диспатчинг

```
add(left, right):
    1. left_dual, right_dual = _dual_number(left), _dual_number(right)
       if оба дуальные: return left_dual.add(right_dual)
       # Компонентное сложение дуальных чисел:
       # (x + v·ε) ⊕ (y + u·ε) = (x⊕y) + (v⊕u)·ε

    2. left_share, right_share = _fullness_share(left), _fullness_share(right)
       if оба доли: return left_share.add(right_share)

    3. Если хотя бы один — ComplexFullness:
       return Π_z

    4. left_complex, right_complex = _complex_state(left), _complex_state(right)
       if оба комплексные: return left_complex.add(right_complex)
       # Компонентное сложение:
       # (x₁ + i·y₁) ⊕ (x₂ + i·y₂) = (x₁+x₂) + i·(y₁+y₂)

    5. left_ratio, right_ratio = _participation_ratio(left), _participation_ratio(right)
       if оба отношения: return left_ratio.add(right_ratio)

    6. Если хотя бы один — Π:
       return Π

    7. Фоллбэк:
       return float(left) + float(right)
```

### 2.3. U-умножение — мультиметод-диспатчинг

```
multiply(left, right):
    1. left_dual, right_dual = _dual_number(left), _dual_number(right)
       if оба дуальные: return left_dual.multiply(right_dual)
       # (x + v·ε) ⊗ (y + u·ε) = (x⊗y) + (x⊗u ⊕ y⊗v)·ε

    2. left_share, right_share = _fullness_share(left), _fullness_share(right)
       if оба доли: return left_share.multiply(right_share)

    3. Если хотя бы один — ComplexFullness:
       return Π_z

    4. left_complex, right_complex = _complex_state(left), _complex_state(right)
       if оба комплексные: return left_complex.multiply(right_complex)
       # (x₁ + i·y₁) ⊗ (x₂ + i·y₂) = (x₁x₂ - y₁y₂) + i·(x₁y₂ + x₂y₁)

    5. left_ratio, right_ratio = _participation_ratio(left), _participation_ratio(right)
       if оба отношения: return left_ratio.multiply(right_ratio)

    6. Если хотя бы один — Ω:
       return Ω

    7. Если хотя бы один — Π:
       return Π

    8. Фоллбэк:
       return float(left) * float(right)
```

### 2.4. Что реализовано

- ✅ `add()` — универсальное U-сложение с мультиметод-диспатчингом
- ✅ `multiply()` — универсальное U-умножение с мультиметод-диспатчингом
- ✅ Все 7 типов состояний поддерживают сложение и умножение
- ✅ Π и Ω обрабатываются как особые элементы

## III. Зачем

Сложение и умножение — базовые связи Носителя:
- **Сложение** — горизонтальная связь (накопление в одном ряду)
- **Умножение** — вертикальная связь (пересечение рядов)

Без связей нет:
- Арифметики на Носителе
- Комплексных вычислений
- Дуальной динамики
- Масштабной инвариантности
