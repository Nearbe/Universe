# 25. Строение Чисел

## I. Теория

p-адику ([24](24_p-адические_числа.md)) дала топология. Теперь — арифметика внутри системы. Из чего состоят числа?

### Атом Системы

Простое число — то, что не разлагается на множители внутри системы.
В мире, где главный оператор `: Ω = D(a)`, класс **D(Id)** — это единственный Атом.

- Все состояния вида `D(a)` — составные.
- Все степени `Dⁿ(Id)` при `n > Id` — составные.
- **D(Id)** — единственное простое, связанное с Актом.

Хребет ([06](06_Степени_двойки.md)) построен на одном Атоме. Всё остальное — его производные.

### НОД и НОК (Меры Близости)

| Мера | Определение | Для степеней |
|---|---|---|
| **НОД(a, b)** | Общий предок. Максимальный масштаб, содержащийся в обоих. | `НОД(Dⁿ, Dᵐ) = D^{min(n,m)}(Id)` |
| **НОК(a, b)** | Общий потомок. Минимальный масштаб, вмещающий оба. | `НОК(Dⁿ, Dᵐ) = D^{max(n,m)}(Id)` |

При Ветвлении (`: Ω`) оба масштабируются синхронно:

```
НОД(D(a), D(b)) = D(НОД(a, b))
```

Родственные связи сохраняются при удвоении.

### Сравнения по Модулю

```
a ≡ b (mod m)
```

означает: **a** и **b** неотличимы на масштабе **m**.

При Ветвлении:

```
a : Ω ≡ b : Ω (mod D(m))  ⟺  a ≡ b (mod m)
```

Ветвление удваивает модуль неразличимости. То, что было одинаковым, остаётся одинаковым, но «зерно» сравнения становится крупнее.

### Двоичная Адресация

Остатки от деления на `Dⁿ(Id)` — это **двоичные слова длины n**.
Ветвление (`: Ω`) действует на них как сдвиг влево.

Это значит: каждое число в системе — это **адрес** в древе фрактала.
Соленоид ([23](23_Соленоид.md)) — это как раз предел этих адресов при `n → ∞`.

### Полный набор операций

| Операция | Формула |
|---|---|
| **Атом** | `D(Id)` — единственный простой |
| **НОД** | `НОД(Dⁿ, Dᵐ) = D^{min(n,m)}(Id)` |
| **НОК** | `НОК(Dⁿ, Dᵐ) = D^{max(n,m)}(Id)` |
| **НОД при ветвл.** | `НОД(D(a), D(b)) = D(НОД(a,b))` |
| **НОК при ветвл.** | `НОК(D(a), D(b)) = D(НОК(a,b))` |
| **Сравнение** | `a ≡ b (mod m) ⟺ a : Ω ≡ b : Ω (mod D(m))` |
| **Адрес** | `a mod Dⁿ(Id)` — двоичное слово длины n |

### II.2. Арифметические операторы (algebra/)

Все операторы реализуют паттерн: dispatch через `_type` helper → типоспецифичный метод.

| Функция | Файл | Типы | Поведение |
|---------|------|------|-----------|
| `branch(value)` | `algebra/branch.py` | Все states + scalar | `D(a) = a : Ω`. Scalar: `scalar * D_ID`. Π → Π |
| `compress(value)` | `algebra/compress.py` | Все states + scalar | `H(a) = a : D(Id)`. Scalar: `scalar / D_ID`. Π → Π |
| `divide(n, d)` | `algebra/divide.py` | Все states + scalar | `n : d`. Ω-правила: `a:Ω=D(a)`, `Ω:b=Ω`, `Ω:Ω=Ω`, `Π:Ω=Π`, `Π:D=Π`, `Π:Π=Id` |
| `add(l, r)` | `algebra/add.py` | Все states + scalar | `l ⊕ r`. Π-замкнутость: `x ⊕ Π = Π` |
| `multiply(l, r)` | `algebra/multiply.py` | Все states + scalar | `l ⊗ r`. Ω-аннигиляция: `Ω*x = Ω`. Π-замкнутость |
| `power(base, exp)` | `algebra/power.py` | SpineLevel, DualNumber, scalar | `base^exp`. Ω-экспонента → Id. Π-экспонента → Π |

## III. Зачем

**Инженерная аргументация.**

Документ определяет арифметическую подсистему проекта — слой, на котором все остальные модули строят свои вычисления. Без этого слоя нет единой модели числовой структуры, и каждый модуль реализует `branch`/`compress`/`divide` независимо.

Ключевые инженерные зависимости:

1. **`SpineLevel`** — фундаментальная единица масштаба. Все модули (`calculus`, `combinatorics`, `renormalization`) ссылаются на `SpineLevel.depth` для определения глубины рекурсии.
2. **`ParticipationRatio`** — типизированная замена `float` для рациональных отношений. Используется когда критично сохранять numerator/denominator раздельно (branch/compression на отношениях).
3. **`DualNumber`** — автоматическое дифференцирование через дуальные числа. `velocity` — это коэффициент при `ε`, который несёт производную. Используется в `calculus` для `derivative()`.
4. **`ComplexState`** — двумерные потоки для `renormalization` и `sweep`. `norm_squared` и `rotate` критичны для RG-анализа.
5. **`FullnessShare`** — типизированная доля от Π. Используется когда нужно сохранять отношение `a : Π` без коллапса в float.
6. **`divide` dispatch** — единая точка входа для всех операций деления. Каждый `_type` helper проверяет тип и делегирует. Это паттерн, который должен соблюдаться при добавлении новых типов.
7. **`number_structure`** — реализует арифметику чисел из оригинала: `system_atom`, `spine_gcd/lcm`, `congruent`, `binary_address`. Это слой между `SpineLevel` и `combinatorics`.
8. **`combinatorics`** — все функции сводятся к `factorial` и `paths_count`. `BranchingLevel` связывает combinatorics с `SpineLevel`.
