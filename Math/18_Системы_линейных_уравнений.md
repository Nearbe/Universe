# 18. Системы Линейных Уравнений

## I. Теория

Одно уравнение — баланс одной степени свободы. Система **𝕌** многомерна: множество связанных состояний.

### Матричная Форма

```
M · x⃗ = b⃗
```

Где:
- **x⃗** — вектор неизвестных (состояния на разных уровнях)
- **b⃗** — вектор целевых состояний
- **M** — матрица связей (карта взаимодействий)

### Решение

При `det M ∉ Ω`:

```
x⃗ = M⁻¹ · b⃗
```

Решение восстанавливает исходные состояния по известным связям и результатам.

### Масштабная Инвариантность

Применим Ветвление (`: Ω`) ко всей системе:

```
D(M) · x⃗ = D(b⃗)
```

Решение **не меняется**:

```
x⃗ = M⁻¹ · b⃗ = D(M)⁻¹ · D(b⃗)
```

Система уравнений фрактальна. Истина (решение **x⃗**) одинакова на любом масштабе. Меняются коэффициенты (**M** и **b⃗**), но отношение между ними сохраняет **x⃗** неизменным.

### Система с Оператором D

Если уравнения содержат явное Ветвление:

```
(D(A) ⊕ B) · x⃗ = b⃗
```

Решение: `x⃗ = (D(A) ⊕ B)⁻¹ · b⃗` (при ненулевом определителе).

### Вырожденный Случай (det M = Ω)

Это не ошибка. Это **схлопывание степеней свободы**.
Когда `det M = Ω`, несколько переменных оказываются жёстко связаны друг с другом. Система теряет независимость.
Это состояние, когда несколько частей фрактала «слипаются» в одну структуру.

## III. Зачем

**Инженерная аргументация:**

1. **Многомерный аналог линейного уравнения.** `LinearSystem` — это обобщение `LinearEquation` на матрицу M. Когда M скалярный (1×1), `solve_linear_system` эквивалентен `solve_linear_equation`.
2. **Масштабная инвариантность — проверяемое свойство.** `LinearSystem.branch()` и `LinearSystem.compress()` позволяют сравнить решения до и после трансформации. Тест `test_branching_system_preserves_solution_truth` подтверждает: `M⁻¹·b = D(M)⁻¹·D(b)` на практике.
3. **`operator_matrix()` — интерфейс для нелинейных систем.** Квадратное уравнение и другие нелинейные задачи строят систему вида `(D(A) ⊕ B)·x = b`. Этот метод — единственный путь создать такую матрицу из компонентов.
4. **Вырожденный случай — семантическое различение.** `COUPLED_STRUCTURE` ≠ `IMPOSSIBLE_REALIZATION`. Система различает "переменные связаны" (det=Ω) от "уравнения противоречат" (в контексте линейного fallback). Это необходимо для iterative execution model: coupled structure может потребовать регуляризации, impossible — требует остановки.
5. **Без внешних зависимостей.** `CoreMatrix`, `CoreVector`, `matmul`, `determinant`, `mat_vec` — полная линейная алгебра без numpy/scipy. Это определяет ограничения: float64-арифметика, O(n³) complexity, no BLAS.
